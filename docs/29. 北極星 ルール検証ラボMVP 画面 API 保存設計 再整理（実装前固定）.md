## 3. MVPで通す1本の標準フロー
1. 北極星で自然言語ルールを入力し、ルールドラフトを保存する  
2. 北極星で自然言語から Pine Script を生成する  
3. 生成 Pine を TradingView に貼り、Strategy Tester で一次検証する  
4. TradingView から検証CSVをエクスポートする  
5. 北極星へ CSV を取り込む  
6. 北極星で検証レポートを表示し、履歴として保存する

### 3.1 責務分担（固定）
- TradingView: 表示・監視・一次検証（実行基盤）
- 北極星: 自然言語変換・履歴保存・レポート表示

北極星内で独自バックテストエンジンをMVPで実装する前提にはしない。

## 4. 画面設計（MVP最小）
## 4.1 画面一覧
1. ルール入力画面（`/strategy-lab`）  
2. Pine変換結果確認画面（`/strategy-lab/:versionId` 相当。MVPでは同一画面内タブでも可）  
3. CSV取込画面（`/backtests/:backtestId/import` 相当）  
4. 検証レポート表示画面（`/backtests/:backtestId`）  
5. 検証履歴一覧画面（任意だが推奨。`/backtests`）

## 4.2 各画面の役割
### 4.2.1 ルール入力画面
- 目的: 自然言語ルールの入力と保存
- 主な入力:
  - strategy_name
  - natural_language_spec
  - target_market
  - target_timeframe
  - backtest_period_from/to
  - commission/slippage（任意）
- 主な表示:
  - 入力内容
  - 入力制約ヒント
  - バリデーションエラー
- 遷移先: Pine変換結果確認画面

### 4.2.2 Pine変換結果確認画面
- 目的: normalized_rule_json / warnings / assumptions / generated Pine の確認
- 主な入力:
  - 変換実行トリガー（生成・再生成）
- 主な表示:
  - normalized_rule_json
  - warnings / assumptions
  - generated Pine
  - TradingViewで一次検証する手順
- 遷移先:
  - TradingView一次検証（外部）
  - CSV取込画面（北極星へ戻る）

### 4.2.3 CSV取込画面
- 目的: TradingView出力CSVのアップロードと parse/normalize 実行
- 主な入力:
  - CSVファイル（metrics / trades）
  - backtest run コンテキスト
- 主な表示:
  - import status（queued / parsed / failed）
  - parse error（失敗時）
- 遷移先: 検証レポート表示画面

### 4.2.4 検証レポート表示画面
- 目的: バックテスト結果の確認と保存履歴の再利用
- 主な表示:
  - run header（symbol, timeframe, period）
  - 主要指標（例: 勝率, PF, DD）
  - trade summary
  - AI総評（MVPでは任意）
  - import files
- 遷移先:
  - 検証履歴一覧
  - （将来）比較導線

### 4.2.5 検証履歴一覧画面（任意）
- 目的: run の再参照
- 主な表示:
  - run name / symbol / timeframe / period / status
- 遷移先: 検証レポート表示画面

---

## 5. 画面遷移（MVP）
`ルール入力` -> `Pine変換結果確認` -> `(TradingViewで一次検証)` -> `CSV取込` -> `検証レポート表示` -> `検証履歴一覧(任意)`

注: TradingViewは北極星外部の実行ステップ。北極星画面遷移としては「Pine確認 -> CSV取込」が主導線。

## 6. API設計（MVP最小）
以下は既存 docs（特に `docs/3`, `docs/16`）を踏まえたMVP最小セット。
レスポンス契約は既存方針どおり `{ data, meta, error }` を維持する。

## 6.1 ルールドラフト保存
- Endpoint: `POST /api/strategies`
- 用途: 戦略ルールの親レコード作成
- 同期/非同期: 同期
- 失敗時:
  - `VALIDATION_ERROR`
  - `CONFLICT`（同名・重複ポリシー採用時）

## 6.2 ルールバージョン保存
- Endpoint: `POST /api/strategies/:strategyId/versions`
- 用途: 自然言語仕様と対象期間・時間足などの保存
- 同期/非同期: 同期
- 失敗時:
  - `VALIDATION_ERROR`
  - `NOT_FOUND`

## 6.3 自然言語->Pine 変換実行
- Endpoint: `POST /api/strategy-versions/:versionId/pine/generate`
- 用途: normalized_rule_json 生成 + Pine生成
- 同期/非同期:
  - MVP初期は同期応答を許容
  - 生成遅延が目立つ場合は job 化（後段）
- 失敗時:
  - `VALIDATION_ERROR`（入力制約外）
  - `AI_GENERATION_FAILED`
  - `PINE_GENERATION_FAILED`

## 6.4 生成Pine取得
- Endpoint: `GET /api/strategy-versions/:versionId/pine`
- 用途: 生成済みPine・メタ情報確認
- 同期/非同期: 同期
- 失敗時:
  - `NOT_FOUND`

## 6.5 backtest run 作成（CSV取込前親レコード）
- Endpoint: `POST /api/backtests`
- 用途: run コンテキスト作成
- 同期/非同期: 同期
- 失敗時:
  - `VALIDATION_ERROR`
  - `CONFLICT`（version/pine不整合）

## 6.6 CSV取込
- Endpoint: `POST /api/backtests/:backtestId/imports`
- 用途: CSVアップロード保存 + parse 起動
- 同期/非同期:
  - uploadは同期
  - parseは非同期または準同期（実装容易な方で開始）
- 失敗時:
  - `UNSUPPORTED_FILE_TYPE`
  - `IMPORT_PARSE_ERROR`
  - `NOT_FOUND`

## 6.7 取込ファイル状態確認
- Endpoint: `GET /api/backtests/:backtestId/imports`
- 用途: parse status, error log の確認

## 6.8 検証レポート取得
- Endpoint: `GET /api/backtests/:backtestId`
- 用途: metrics/trade summary/import files/総評の表示

## 6.9 検証履歴一覧取得
- Endpoint: `GET /api/backtests`
- 用途: 履歴一覧表示（任意画面）

## 8. MVP入力制約（自然言語->Pine）
「なんでも書けば変換できる」は前提にしない。以下をMVP入力制約として固定する。

1. 記述言語
- 日本語入力を基本対応（MVP）
- 英語混在はベストエフォート（保証外）

2. 対応時間足
- 日足（`D`）を主対象
- 追加時間足は後段

3. 売買ルールの複雑度
- 単純な long_only 条件を主対象
- entry / exit はAND条件中心
- 高度なポジション管理（分割利確、複雑トレーリング、多段条件）はMVP対象外

4. 指標セット
- 移動平均 / RSI / 出来高系など代表指標に限定
- 独自関数・高度な複合条件は後段

5. 出力品質
- warnings / assumptions を必ず返す
- unsupported な記述は黙って解釈せず警告を返す
