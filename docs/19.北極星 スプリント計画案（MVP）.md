# 北極星 スプリント計画案（MVP）

更新日: 2026-03-09

## 1. このドキュメントの目的

本ドキュメントは、株価評価AIツール「北極星」のMVP開発に向けて、これまで整理してきた仕様・設計・実装チケットをもとに、**現実的なスプリント単位の開発計画** を定義するものである。

北極星のMVPでは、以下の3本の縦導線を成立させることが重要である。

- TradingView alert → 要約 → ホーム
    
- 銘柄詳細 → AI論点 → ノート
    
- 自然言語 → Pine → CSV取込 → 総評
    

本資料では、これらを壊さずに段階的に成立させるため、スプリントごとの目標、対象チケット、完了条件、リスク、見直しポイントを整理する。

---

## 2. スプリント計画の前提

## 2.1 想定する進め方

MVPでは、仕様をすべて横に広げて同時に作るのではなく、**縦導線が動く順** に段階的に積み上げる。

想定順序:

1. 基盤を作る
    
2. webhook 受信と alert 要約を通す
    
3. ホームを成立させる
    
4. 銘柄研究を成立させる
    
5. 自然言語→Pine を成立させる
    
6. CSV取込と backtest 表示を成立させる
    
7. 比較 / 総評で仕上げる
    

---

## 2.2 スプリント期間の想定

MVP計画では、**1スプリント = 1〜2週間程度** を想定する。

本資料では、5スプリント構成で整理する。

- Sprint 1: 基盤 + webhook 受信の最初の成立
    
- Sprint 2: alert 要約 + ホーム成立
    
- Sprint 3: 銘柄研究 + ノート
    
- Sprint 4: ルール検証 + Pine
    
- Sprint 5: CSV取込 + backtest 詳細 + 比較
    

---

## 2.3 進行管理の考え方

各スプリントで重視すること:

- スプリントの最後に「動くもの」がある
    
- 次スプリントの前提が整う
    
- 失敗しやすい箇所を早めに表面化する
    

特に北極星では、以下を早期に確認する必要がある。

- TradingView webhook の実疎通
    
- symbol 解決の安定性
    
- AI schema の安定性
    
- Pine 生成の最低限の品質
    
- CSV parser の現実運用耐性
    

---

## 3. 全体マイルストーン

### M1

TradingView alert を受信し、北極星側に alert_event が保存される

### M2

alert 起点の要約が生成され、ホーム画面で確認できる

### M3

銘柄詳細で references / AI論点 / ノートが扱える

### M4

自然言語から Pine を生成し、TradingView に貼れる

### M5

backtest CSV を取り込み、検証結果を見て比較できる

# 14. 結論

北極星MVPのスプリント計画では、  
**仕様を横並びで消化するのではなく、価値のある縦導線を段階的に成立させる** ことを基本方針とする。

そのため、以下の順番で進める。

- Sprint 1: 基盤 + webhook
    
- Sprint 2: 要約 + ホーム
    
- Sprint 3: 銘柄研究 + ノート
    
- Sprint 4: 自然言語→Pine
    
- Sprint 5: CSV取込 + backtest 比較
    

この流れにより、北極星MVPを **早く動く形で確認しながら、壊れやすい部分を先に固める** 進め方を実現する。
