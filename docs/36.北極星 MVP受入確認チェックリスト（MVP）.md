# 北極星 MVP受入確認チェックリスト（MVP）

更新日: 2026-05-07

## 1. 目的
- MVP完了前に確認すべき項目を一覧化する
- 受入確認の結果を `OK / NG / 要確認 / MVP後` で記録できるようにする
- 確認対象は「個人用実運用ツールとして最低限回るか」

## 2. MVP完了判定
MVP完了条件:

- main 最新で backend / frontend build が通る
- required checks が通る
- README / walkthrough に沿って起動できる
- Home / SymbolDetail / Comparison が動く
- watchlist / positions の実データ管理が動く
- TradingView webhook 実送信が成立している
- Alert summary が生成・表示される
- Rule Lab から Pine 生成できる
- TradingView CSV import ができる
- Backtest Detail / AI総評 / 比較が確認できる
- AI出力で `content空 / reasoningのみ / length打ち切り` が再発していない
- references不足時の表示が破綻しない
- secret / token / shared_secret が docs に残っていない
- 主要な既知課題が docs に記録されている

## 4. 判定基準
- OK: MVP完了判定に支障なし
- 要確認: 使えるが、補足確認または手動確認が必要
- NG: MVP完了前に修正が必要
- MVP後: MVP完了を阻害しない既知課題

## 5. MVP完了前に必ず見る項目
必須チェック:

- 起動
- Home
- watchlist / positions
- SymbolDetail
- Comparison
- TradingView webhook
- Alert summary
- Rule Lab / Pine
- CSV import / Backtest
- AI品質
- secret 非露出

## 6. MVP後に回してよい項目
| 項目 | 理由 | 判定 |
|---|---|---|
| Performance Summary 日本語ヘッダー対応 | 現行 parser の対象外だが MVP 主導線は英語ヘッダーで成立している | MVP後 |
| disclosure / earnings の供給改善 | diagnostics で理由追跡は可能だが供給改善は別タスク | MVP後 |
| references あり銘柄での追加 TradingView 実送信 | 実送信 1 件の疎通確認は完了済み | MVP後 |
| snapshot stale / market_status unknown 整理 | 表示改善の余地だが MVP ブロッカーではない | MVP後 |
| browser-based E2E 導入 | 現状は既存 E2E と手動確認で運用している | MVP後 |
| 複数ポートフォリオ UI | 個人利用 MVP 範囲外 | MVP後 |
| transaction 履歴編集 UI | positions の最小運用には不要 | MVP後 |
| 非日本銘柄の本格対応 | 一部動作はするが本格運用は対象外 | MVP後 |
| StrategyLab 文言テスト追加 | 品質補強として有用だが MVP 完了を阻害しない | MVP後 |
| docs 全体の文字化け完全棚卸し | 主要 runbook 正常化を優先し、全量棚卸しは別タスク | MVP後 |
