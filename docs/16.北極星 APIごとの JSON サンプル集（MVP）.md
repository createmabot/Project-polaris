# 北極星 APIごとの JSON サンプル集（MVP）

更新日: 2026-03-09

## 1. このドキュメントの目的

本ドキュメントは、株価評価AIツール「北極星」のMVPにおいて、主要APIの **request / response の JSON サンプル** を整理するものである。

これまでに、以下の設計を整理してきた。

- データモデル設計
    
- API / ユースケース単位の入出力設計
    
- webhook受信仕様
    
- AI要約フロー
    
- external references 収集
    
- 自然言語→Pine変換
    
- CSVパース
    
- バックテスト比較
    
- MVPタスク分解
    

本資料は、これらを **実装時・フロント接続時に使える具体例** に落とすことを目的とする。

---

## 2. 共通レスポンス形式

北極星の API は、原則として以下の共通形式を採用する。

```json
{
  "data": {},
  "meta": {},
  "error": null
}
```

エラー時は以下を基本形とする。

```json
{
  "data": null,
  "meta": {
    "request_id": "req_xxx"
  },
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "symbol_id is required"
  }
}
```

- `result_summary` は runtime で schema 検証する（`schema_version = "1.0"` と `summary_kind` を含む最小 shape）。
- `artifact_pointer` は runtime で schema 検証する（`type` / `execution_id` / `path` 必須）。

## 14. 実装上の注意

### 14.1 サンプルは固定仕様ではなく MVP 基準の代表例

実装時には、optional 項目や null を許容する設計にする。

### 14.2 structured_json は必ず schema バリデーションを通す

特に AI 出力は保存前バリデーションが重要。

### 14.3 数値と文字列の型を崩さない

- price, rate, quantity は number
    
- id, code, label は string
    

### 14.4 日付文字列は ISO 8601 に寄せる

特に webhook / AI job / summary / backtest で統一する。
