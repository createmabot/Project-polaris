# 北極星 システム構成 現行仕様

更新日: 2026-05-26
分類: 仕様書

## 1. 目的

本資料は、北極星の現行システム構成を仕様として整理する。作業履歴やPR経緯ではなく、実装判断時に確認する構成要素、責務、境界を固定する。

## 2. 構成要素

- Frontend: Vite + React + TypeScript。画面表示、read-only履歴確認、CSV import、AI summary generation の操作導線を担う。internal backtest start / report conversion の UI 導線は閉じている。
- Backend: Fastify + TypeScript。REST API、validation、AI summary generation、CSV import、Backtest report、Symbol Strategy Application を担う。
- DB: PostgreSQL + Prisma。schema と migrations を正本として扱う。
- Queue / cache: Redis / BullMQ。AI jobs など非同期処理の状態管理に使う。
- External / local integrations: TradingView webhook / CSV、local LLM または OpenAI API provider。

## 3. 責務境界

- TradingView は chart / alert / CSV source として扱う。北極星は TradingView の実行結果を CSV import report として保存・確認する。
- internal backtest は historical legacy として扱う。Stage 2C cleanup 後は execution backend / worker / route / table は存在せず、既存 report は Backtest snapshot ベースで read-only 表示する。
- AI provider は `ai_jobs` / `ai_summaries` と provider boundary を通して扱う。secret 実値は docs / log / PR に残さない。
- Frontend は application / run / report の表示とユーザー操作を行う。永続化判断、重複制御、status 更新は Backend / DB 側で行う。
- Prisma schema と migrations は DB 構造の正本であり、本仕様書は概念と読み方を補助する。

## 4. 主要画面との関係

- Home / SideRail: 日次確認と監視・保有銘柄の入口。
- SymbolDetail: 銘柄起点の application 概要、latest run / report、CSV import、historical report read-only 確認、ApplicationDetail への入口。
- ApplicationDetail: application 単位の run / report 履歴。実行操作は持たない read-only foundation。
- StrategyDetail / StrategyVersionDetail / StrategyLab: strategy 作成・保存・適用・検証の入口。
- BacktestDetail / BacktestComparisonDetail: report detail と比較補助。

## 5. 非対象

- OpenAPI document の自動生成。
- metrics normalization table。
- comparison entity の新規作成。
- hard delete。
- docs への実 secret / token / local path 記載。

## 投資カレンダー構成

Backend は `InvestmentCalendarEvent` を DB に保存し、Home / SymbolDetail へ read API と manual refresh API を提供する。provider boundary は `stub|public` を持つが、required check は stub / fake に限定する。raw external response / endpoint 実値 / stack trace は保存・表示しない。

Frontend は Home と SymbolDetail に compact list と refresh button を表示する。TradingView chart widget は SymbolDetail から削除し、TradingView CSV import / Pine validation flow は維持する。
