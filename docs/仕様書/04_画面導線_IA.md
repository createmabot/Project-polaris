# 北極星 screen / IA 現行仕様

更新日: 2026-05-13
分類: 仕様書

## 1. 目的

本資料は、現行の主要画面責務と導線を整理する仕様書である。画面実装の経緯は `docs/53`、IA方針の履歴は `docs/47` を参照する。

## 2. 主要画面責務

| 画面 | 現行責務 |
|---|---|
| Home | 日次確認、watchlist / positions / market overview / latest summary の入口。 |
| SideRail | 監視銘柄 / 保有銘柄の主要導線と CRUD modal。旧 `/watchlist` / `/positions` は補助 route。 |
| SymbolDetail | 銘柄単位の日次確認、投資カレンダー、AI論点、latest alert、Symbol Strategy Application / 検証結果、関連参照、Research Note の入口。 |
| StrategyDetail | strategy の説明、version / applied symbols / related reports の read-only 確認。 |
| StrategyVersionDetail | strategy version、Pine、validation、internal backtest、application 起点の確認。 |
| StrategyLab | natural language strategy 作成、LLM strategy proposal 候補の確認、Pine生成、保存、検証の作業画面。 |
| ApplicationDetail | Symbol Strategy Application 単位の run / report history、filter / pagination。 |
| BacktestDetail | 個別 Backtest report detail、同一 application 関連 report、metrics comparison helper、AI summary / artifact 表示。 |
| BacktestComparisonDetail | 保存済み pairwise comparison の再訪画面。本格 comparison route 候補。 |

## 3. 導線方針

- 監視銘柄 / 保有銘柄の主要導線は SideRail に寄せる。
- SymbolDetail は application の概要と入口に留め、run / report history は ApplicationDetail へ分離する。
- SymbolDetail は日次確認ゾーン（銘柄概要、投資カレンダー、最新AI論点カード、最新アラート）を上部に置き、検証・調査ゾーン（ストラテジー / 検証結果、関連参照情報、Research Note）を下部に置く。
- BacktestDetail は個別 report detail と近接比較補助に留める。
- ApplicationDetail は application-scoped history の受け皿とする。
- BacktestComparisonDetail は保存済み comparison の再訪画面として維持し、本格比較画面化は後続判断とする。
- `/watchlist` と `/positions` は移行期の補助 route であり、主要導線ではない。

## 4. Post-P3 acceptance の主要導線

- Home / SideRail から SymbolDetail へ進む。
- SymbolDetail から StrategyDetail / ApplicationDetail / BacktestDetail へ進む。
- StrategyDetail から StrategyVersionDetail / BacktestDetail へ進む。
- ApplicationDetail から BacktestDetail へ進む。
- BacktestDetail から related reports / comparison helper / BacktestComparisonDetail を確認する。
- StrategyLab から strategy / version / Pine / CSV import / BacktestDetail の一巡導線を確認する。
- LLM strategy proposal は StrategyLab 内の一時候補から natural language spec へ反映する導線とし、proposal 選択だけで保存や Pine 生成は行わない。
- Home / SideRail の watchlist / positions CRUD modal を確認する。

詳細な release acceptance checklist は `docs/運用ドキュメント/10_release_acceptance_checklist.md` を参照する。

## 5. Symbol centered strategy workflow 方針

SymbolDetail は、銘柄単位の日次確認だけでなく「この銘柄で戦略を考える」「この銘柄に適用した戦略を改善する」入口も担う。ただし StrategyLab / StrategyVersionDetail の既存責務は維持し、SymbolDetail は context handoff と明示 action の入口に留める。

初回は DB / Prisma schema 変更なしで、route query / return path による context handoff を採用する。SymbolDetail から StrategyLab / StrategyVersionDetail へ渡す context は、`symbol_id`、`symbol_code`、symbol display name、market、default timeframe、return path、必要に応じて `application_id` / `strategy_id` / `strategy_version_id` / mode を含める。context は画面表示と prefill に使うが、永続的な lineage とは扱わない。

Phase 1 では、SymbolDetail の `ストラテジー / 検証結果` section に `この銘柄でストラテジー提案` action を置き、StrategyLab へ symbol context を渡す。StrategyLab は context banner を表示し、proposal input の market / symbol code / timeframe を prefill する。proposal candidate selection は従来どおり title / natural language spec への反映だけであり、Pine generation、strategy save、backtest、AI summary は自動起動しない。

Phase 2 では、StrategyLab が symbol context 付きで起動された場合だけ、strategy / version 保存後に `この銘柄に適用` と `銘柄ページへ戻る` の明示 CTA を出す。適用は既存の symbol strategy application 作成 API を使い、active duplicate conflict は既存 error として扱う。保存直後の自動適用は行わない。

Phase 3 では、SymbolDetail の application row から `改善版を作る` action を追加し、既存 strategy version clone flow へつなぐ。clone 後の StrategyVersionDetail には improvement context banner を表示し、自然言語ルール編集、Pine 再生成、TradingView 検証結果を反映した Pine 修正再生成は既存導線を使う。

Phase 4 では、改善版の StrategyVersionDetail に `この銘柄に改善版を適用` と `元の銘柄ページへ戻る` CTA を置く。改善版適用も既存 application 作成 API の明示操作に限定する。旧 application は active のまま残し、run / report history を保持する。`改善版を適用して旧 application を archive`、新旧 application lineage、proposal candidate から StrategyVersion への永続 relation は後続判断とする。

BacktestDetail handoff phase では、`symbol_strategy_application` を持つ BacktestDetail にだけ `この検証結果をもとに改善版を作る` action を置く。クリック時だけ既存 `POST /api/strategy-versions/:versionId/clone` を呼び、clone 後の StrategyVersionDetail へ `source_backtest_id` と `/backtests/:backtestId` return path を含む improvement context を渡す。StrategyVersionDetail は元 backtest を read-only context として表示し、改善メモを既存 `revision_request` textarea に反映できるが、表示・反映だけで Pine 再生成、backtest、AI summary、apply は起動しない。

維持する境界:

- SymbolDetail 表示だけで proposal generation、AI / provider call、clone、apply を起動しない。
- BacktestDetail 表示だけで clone、Pine generation、save、backtest、AI summary generation、apply を起動しない。
- StrategyVersionDetail の `source_backtest_id` は read-only context であり、fetch 失敗時も base version 表示と rule / Pine 操作を壊さない。
- proposal selection から Pine generation、strategy save、backtest、AI summary を自動実行しない。
- 改善版適用時に旧 application を自動 archive しない。
- internal backtest 新規実行 UI と TradingView embedded chart widget は復活させない。
- Web search / deep research job、TradingView compile automation、scheduled job は追加しない。
- raw prompt、raw provider response、raw Codex output、endpoint、model 実値、secret、local path、stack trace は UI / API / docs / PR に出さない。

## 投資カレンダー導線

Home は watchlist / positions 対象銘柄と market-level event をまとめた投資カレンダーを表示する。SymbolDetail は対象銘柄に関係する投資カレンダーを Home と同じ compact calendar grid で表示し、手動更新ボタンを持つ。SymbolDetail の月送りは frontend local state の表示切替だけで、API fetch / provider refresh / URL query 変更は行わない。TradingView chart widget は SymbolDetail から削除し、TradingView は外部での chart / Pine 検証と CSV import の主導線として扱う。
