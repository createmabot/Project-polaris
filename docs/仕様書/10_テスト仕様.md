# 北極星 テスト仕様 現行仕様

更新日: 2026-05-24
分類: 仕様書

## 1. 目的

本資料は、現行テスト方針の入口である。個別テストの詳細は test files と Playwright spec を正本として扱う。

## 2. backend tests

- route / API behavior は backend e2e tests で固定する。
- Symbol Strategy Application / Backtest import / AI summary / internal backtest は、該当 e2e test を優先して確認する。
- validation error、pagination、status transition、idempotent report conversion は backend tests で確認する。
- Strategy proposal benchmark result recording は、real provider 依存 test ではなく、stub / fake data による schema、sanitization、output path validation を確認する。
- Strategy proposal Codex CLI manual JSON import は、real Codex CLI 依存 test ではなく、fake `strategy_proposal_candidates` JSON fixture で prompt endpoint、import validation、candidate count、history 保存、selection 非回帰、raw output 非露出を確認する。
- Strategy proposal Codex CLI Web検索付き prompt option は、real Codex CLI / Web search 依存 test ではなく、`web_search_prompt=false` では既存 prompt のまま、`web_search_prompt=true` では Web 検索利用時の注意文だけが追加され、import endpoint / schema / provider name / selection flow が変わらないことを確認する。
- Strategy proposal history full management は、provider / status / selected / search / pagination の backend behavior、safe list serialization、detail / selection 非回帰、provider quality trend 非回帰を fake history data で確認する。
- Strategy proposal sanitized provider event log persistence は、real provider 依存ではなく fake / mock data で provider generation success / failure、local_llm retry、proposal generation rate limited、Codex CLI import success / failure / rate limited、event read API pagination / filter、raw prompt / raw provider response / raw Codex output / endpoint / model 実値 / local path / stack trace / user_hint 全文 / candidate free text 非保存を確認する。
- Strategy proposal history soft archive は、new run が active で作成されること、default / `archived=active` list が archived run を除外すること、`archived=archived` / `archived=all` filter、archive / unarchive idempotency、archived detail、archived run からの candidate selection、provider event log 非影響、hard delete 不在、raw leakage boundary を backend tests で確認する。
- Pine generation LLM-first は、real `local_llm` / `openai_api` / TradingView compile に依存せず、fake provider output と deterministic baseline / fallback fixture で確認する。
- Pine generation backend tests は、StrategyVersion generate context、`JP_STOCK|US_STOCK` と `D|1D|4H|1H` の扱い、unsupported value の warning / assumption、generator -> reviewer -> repair pipeline、LLM output の最小 validation、retryable invalid output / reviewer issue の bounded repair、`repair_attempts` / `invalid_reason_codes`、provider failure の sanitized failure を確認する。
- Pine generation tests では deterministic reviewer が明らかな syntax / style / safety issue を検出すること、AI reviewer provider boundary が structured issue だけを返すこと、既存 normalization / validation が維持されることを fake / deterministic data で確認する。
- Pine generation tests では raw prompt、raw provider response、raw reviewer response、endpoint、model 実値、secret、local path、stack trace が API response / persisted note / UI に出ないことを確認する。real `local_llm` / OpenAI / TradingView 実体依存 test は required check に入れない。
- Prisma schema validation は `pnpm --filter backend exec prisma validate` を使う。

## 3. frontend tests

- 画面ごとの rendering / interaction intent は page tests で固定する。
- Tailwind class の詳細は原則 test しない。
- route / link / stable text / read-only display / mutation intent を優先する。
- ApplicationDetail は read-only history、SymbolDetail は application 操作導線として分けて確認する。
- StrategyLab の Codex CLI manual JSON import UI は、prompt 作成、JSON paste / file text import、複数候補 card 表示、candidate selection、sanitized error 表示を確認する。real Codex CLI は frontend test の dependency にしない。
- StrategyLab の proposal history management UI は、provider / status / selected filter、search、pagination、empty / loading / error、filtered detail からの candidate selection を確認する。raw prompt / raw provider response / raw Codex output / endpoint / model 実値 / local path / stack trace を UI に出さないことも確認する。
- StrategyLab の proposal history soft archive UI は、default active filter、archive action で active list から隠れること、archived filter で表示されること、unarchive で active に戻ること、all filter、archived detail / selection flow、hard delete UI がないことを確認する。
- StrategyLab / StrategyVersionDetail の Pine generation UI は、market / timeframe 選択肢、TradingView chart timeframe 前提の helper text、LLM-first / validation / bounded repair の sanitized 表示、同期 request 中の frontend-only 概算 progress indicator を確認する。progress step は generator -> reviewer -> repair pipeline の近似表示であり、backend streaming / polling / job / SSE / WebSocket を test dependency にしない。endpoint / model / raw prompt / raw provider response / raw reviewer response は UI に出さない。real TradingView compile、real local_llm、auto Pine、auto save、auto backtest、AI summary 自動生成は test dependency にしない。

## 4. browser smoke

- `pnpm test:e2e:browser` は主要 read-only flow を固定する。
- CRUD / CSV import execution / internal backtest execution / AI summary generation は smoke に含めすぎない。
- Visual regression pilot は `pnpm test:e2e:visual` に分離し、`pnpm test:e2e:browser` と CI required check には含めない。
- 初回 pilot は `ApplicationDetail` の `application summary` stable container 1 箇所に限定する。SymbolDetail を含むその他画面の visual snapshot は未実装である。

## 4-1. optional benchmark

- `pnpm --filter backend strategy-proposal:benchmark` は Strategy proposal の manual / optional benchmark であり、required check には入れない。
- default provider は env に依存しない `stub`。
- `--provider=local_llm` は manual optional であり、local_llm 実体依存 test は required check に入れない。
- `--output=<file>.json` の record output は gitignore 済み local directory に限定し、actual record は commit しない。
- CI で確認する場合は、stub / fake data で sanitized summary schema、raw prompt / raw response / endpoint / model 実値 / local path / user_hint 全文 / candidate 自由文本文が出ないことを確認する。

## 5. docs-only PR

- docs-only PR は最低限 `git diff --check` を確認する。
- changed docs に local path / secret 実値が残っていないことを確認する。
- PR本文は ASCII English only、docs本文は UTF-8 日本語を維持する。
- 実装変更がない場合、build / e2e を無理に実行しない。
- release acceptance / stabilization の docs-only PR では `docs/運用ドキュメント/10_release_acceptance_checklist.md` と README / `docs/0` / `docs/57` / walkthrough の導線が矛盾しないことを確認する。
