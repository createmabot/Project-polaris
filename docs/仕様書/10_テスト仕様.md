# 北極星 テスト仕様 現行仕様

更新日: 2026-05-15
分類: 仕様書

## 1. 目的

本資料は、現行テスト方針の入口である。個別テストの詳細は test files と Playwright spec を正本として扱う。

## 2. backend tests

- route / API behavior は backend e2e tests で固定する。
- Symbol Strategy Application / Backtest import / AI summary / internal backtest は、該当 e2e test を優先して確認する。
- validation error、pagination、status transition、idempotent report conversion は backend tests で確認する。
- Strategy proposal benchmark result recording は、real provider 依存 test ではなく、stub / fake data による schema、sanitization、output path validation を確認する。
- Strategy proposal Codex CLI manual JSON import は、real Codex CLI 依存 test ではなく、fake `strategy_proposal_candidates` JSON fixture で prompt endpoint、import validation、candidate count、history 保存、selection 非回帰、raw output 非露出を確認する。
- Prisma schema validation は `pnpm --filter backend exec prisma validate` を使う。

## 3. frontend tests

- 画面ごとの rendering / interaction intent は page tests で固定する。
- Tailwind class の詳細は原則 test しない。
- route / link / stable text / read-only display / mutation intent を優先する。
- ApplicationDetail は read-only history、SymbolDetail は application 操作導線として分けて確認する。
- StrategyLab の Codex CLI manual JSON import UI は、prompt 作成、JSON paste / file text import、複数候補 card 表示、candidate selection、sanitized error 表示を確認する。real Codex CLI は frontend test の dependency にしない。

## 4. browser smoke

- `pnpm test:e2e:browser` は主要 read-only flow を固定する。
- CRUD / CSV import execution / internal backtest execution / AI summary generation は smoke に含めすぎない。
- Visual regression pilot は `pnpm test:e2e:visual` に分離し、`pnpm test:e2e:browser` と CI required check には含めない。
- 初回 pilot は `ApplicationDetail` の `application summary` stable container 1 箇所に限定する。SymbolDetail を含むその他画面の visual snapshot は未実装である。

## 4-1. optional benchmark

- `pnpm --filter backend strategy-proposal:benchmark` は Strategy proposal の manual / optional benchmark であり、required check には入れない。
- default provider は env に依存しない `stub`。
- `--provider=local_llm` は manual optional であり、local_llm 実体依存 test は required check に入れない。
- `--output=<file>.json` の record output は gitignore 済み local directory に限定し、actual record は commit しない。
- CI で確認する場合は、stub / fake data で sanitized summary schema、raw prompt / raw response / endpoint / model 実値 / local path / user_hint 全文 / candidate 自由文本文が出ないことを確認する。

## 5. docs-only PR

- docs-only PR は最低限 `git diff --check` を確認する。
- changed docs に local path / secret 実値が残っていないことを確認する。
- PR本文は ASCII English only、docs本文は UTF-8 日本語を維持する。
- 実装変更がない場合、build / e2e を無理に実行しない。
- release acceptance / stabilization の docs-only PR では `docs/運用ドキュメント/10_release_acceptance_checklist.md` と README / `docs/0` / `docs/57` / walkthrough の導線が矛盾しないことを確認する。
