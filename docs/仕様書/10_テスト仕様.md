# 北極星 テスト仕様 現行仕様

更新日: 2026-05-13
分類: 仕様書

## 1. 目的

本資料は、現行テスト方針の入口である。個別テストの詳細は test files と Playwright spec を正本として扱う。

## 2. backend tests

- route / API behavior は backend e2e tests で固定する。
- Symbol Strategy Application / Backtest import / AI summary / internal backtest は、該当 e2e test を優先して確認する。
- validation error、pagination、status transition、idempotent report conversion は backend tests で確認する。
- Prisma schema validation は `pnpm --filter backend exec prisma validate` を使う。

## 3. frontend tests

- 画面ごとの rendering / interaction intent は page tests で固定する。
- Tailwind class の詳細は原則 test しない。
- route / link / stable text / read-only display / mutation intent を優先する。
- ApplicationDetail は read-only history、SymbolDetail は application 操作導線として分けて確認する。

## 4. browser smoke

- `pnpm test:e2e:browser` は主要 read-only flow を固定する。
- CRUD / CSV import execution / internal backtest execution / AI summary generation は smoke に含めすぎない。
- Visual regression は pilot 判断済みだが、本導入はまだ行わない。

## 5. docs-only PR

- docs-only PR は最低限 `git diff --check` を確認する。
- changed docs に local path / secret 実値が残っていないことを確認する。
- PR本文は ASCII English only、docs本文は UTF-8 日本語を維持する。
- 実装変更がない場合、build / e2e を無理に実行しない。
