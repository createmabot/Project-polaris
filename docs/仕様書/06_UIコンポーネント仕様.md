# 北極星 UI component 現行仕様

更新日: 2026-05-22
分類: 仕様書

## 1. 目的

本資料は、現行の共通UI componentの責務を整理する仕様書である。実装経緯や未対応画面の履歴は `docs/53` を参照する。

## 2. 現行 component

| component | 責務 | 注意 |
|---|---|---|
| `SectionCard` | section wrapper。画面固有文言は呼び出し側に残す。 | 全面カード化には使わない。 |
| `Surface` | filter block、history card、nested metadata、readable text block の薄い wrapper。 | 業務ロジック、見出し、API、文言は持たない。万能 card 化しない。 |
| `EmptyState` | 空状態の表示。 | 操作や retry 仕様は持たない。 |
| `ErrorState` | 取得失敗や表示不能の説明。 | backend error 契約は持たない。 |
| `LoadingState` | loading 表示。 | polling / skeleton の本格化はしない。 |
| Pine generation progress indicator | StrategyLab / StrategyVersionDetail の Pine generation job status を polling して表示する進捗。 | SSE / WebSocket / streaming ではなく、endpoint / model / raw prompt / raw response / raw reviewer response は表示しない。 |
| `Button` | 操作 button の最小 variant。 | Link 表現は `TextLink`。 |
| `TextLink` | 画面間遷移 link。 | button 操作とは分ける。 |
| `StatusBadge` | status 文字列の badge 表示。 | status enum の正規化はしない。 |
| `KeyValueList` / `KeyValueRow` | key-value 表示。 | JSON dump や table 代替にはしない。 |
| `TextInput` / `TextArea` / `SelectField` | 最小 form field。 | form framework は導入しない。 |
| `ModalShell` | selected modal の薄い wrapper。 | keyboard trap の本格実装は対象外。 |
| `JsonBlock` | read-only JSON 表示。 | JSON diff / collapsible / copy button は対象外。 |
| `FilterGroup` | filter UI の小さな grouping。 | query semantics は画面側に残す。 |
| `PaginationControls` | prev / next pagination。 | DataTable / infinite scroll は対象外。 |
| `InlineNotice` | helper / note / source差の補足。 | toast / validation / alert system ではない。 |

## 3. 共通化しない方針

- DataTable 高機能化、virtual scroll、column resize は行わない。
- form framework は導入しない。
- Modal accessibility framework は導入しない。
- Visual regression 本導入は別フェーズ判断にする。
- 画面固有文言、API呼び出し、業務ロジックは component に閉じ込めない。

## 4. UI/UX production readiness phase 1 の扱い

- Phase 1 では既存 component の責務を変えず、border、shadow、focus、disabled、hover、spacing を揃えて app UI としての密度を改善する。
- `SectionCard` は section 境界と見出し階層を明確にする用途に留め、画面固有の業務判断は持たない。
- `Button` は primary / secondary / danger の視覚差と focus-visible / disabled state を扱うが、link navigation は引き続き `TextLink` に分ける。
- `EmptyState` / `LoadingState` / `ErrorState` / `InlineNotice` は補助状態の表示に限定し、toast system、validation framework にはしない。Pine generation progress indicator の polling は Pine job status endpoint に限定する。
- Home / SideRail / SymbolDetail / StrategyLab の改善で必要になった最小範囲だけを反映し、DataTable、virtual scroll、modal framework、visual regression required check は導入しない。

## 5. UI/UX production readiness phase 2 の扱い

- Phase 2 では ApplicationDetail / BacktestDetail / StrategyDetail / StrategyVersionDetail / BacktestComparisonDetail に既存 component を適用し、page shell、heading、card density、loading / error / empty state の見え方を揃える。
- `AppLayout` / `PageHeader` は検証・履歴系画面にも使うが、各画面の API 呼び出し、mutation、business logic は component に移さない。
- `SectionCard` は画面内の作業単位と read-only 情報のまとまりを示す用途に留める。BacktestDetail / StrategyVersionDetail の全面 component 化は一度に行わず、高頻度 section から小さく進める。
- `KeyValueList` / `StatusBadge` / `TextLink` は既存の検証レポート、strategy version、comparison の情報表示を揃えるために使う。DataTable、高機能 table、virtual scroll、modal framework は引き続き後続判断とする。

## 6. UI/UX production readiness phase 3 の扱い

- Phase 3 では Phase 1 / Phase 2 で実際に複数画面へ重複した surface 表現だけを `Surface` として切り出す。
- `Surface` は `card`、`muted`、`nested`、`readable` の最小 variant に限定し、filter block、proposal card、run / report card、nested metadata、read-only text block の視覚統一だけを担う。
- `Surface` は画面固有文言、API 呼び出し、mutation、status 正規化、pagination、selection、provider logic を持たない。
- DataList / SimpleTable / DataTable、高機能 table、virtual scroll、column resize、modal framework 全面導入、visual regression required check 化は引き続き後続判断とする。

## 7. Core screens compact density pass の扱い

- Core screens compact density pass では、PageHeader / SectionCard / Surface の既定余白を一段詰め、主要画面の list / history / metadata の表示密度を揃える。
- 共通 component は薄い wrapper のままとし、画面固有文言、API 呼び出し、mutation、refresh、provider、job enqueue の責務は持たない。
- DataTable / virtual scroll / column resize / modal framework 全面導入は引き続き後続判断にする。

## 投資カレンダー UI component 方針

投資カレンダーは `SectionCard`、`InlineNotice`、`EmptyState`、既存 button を使う。Home では 1ヶ月分の compact calendar grid を使い、`前月` / `次月` button で frontend local state の表示月だけを切り替える。月送りは API fetch / provider refresh / URL query 変更を伴わない。day cell には短い event chip を積み、chip 内は event type prefix と title 中心にする。provider / source / fetched_at / importance text / stale detail は chip 内に出さず、grid 下部の metadata で確認する。1 日の event が多い場合は最大表示数を制限し、残りは `+N件` として省略する。土日 weekday header / day cell は平日と控えめに区別する。SymbolDetail は銘柄別 event の compact list を維持する。notification、external calendar sync、full calendar library は作らない。
