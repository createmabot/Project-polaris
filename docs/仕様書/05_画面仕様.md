# 北極星 画面仕様 現行仕様

更新日: 2026-05-17
分類: 仕様書

## 1. 目的

本資料は、主要画面の現行責務を仕様として整理する。画面導線全体は `docs/仕様書/04_画面導線_IA.md` を参照する。

## 2. Home / SideRail

- Home は日次確認、summary、主要導線の入口。
- SideRail は監視銘柄 / 保有銘柄の主要導線と CRUD modal を担う。
- Watchlist / Positions の旧管理画面 route は移行期補助として扱い、主要導線は SideRail に寄せる。

## 3. SymbolDetail

- symbol の概要、latest alert / AI論点 / related references を表示する。
- related references はユーザー操作起点の手動再取得ボタンを持つ。表示起点、自動定期取得、batch 起点の refresh は行わない。
- saved application の概要、latest run、latest report、CSV / internal latest pair を表示する。
- server-side filter を使い、application status / report presence / report source / latest run type / latest run status / strategy / strategy version を切り替える。
- application 作成、archive / restore、CSV import、internal backtest start、internal report conversion の操作導線を持つ。
- 既存 strategy 選択は active strategy の検索、件数制限、ページ送りを持ち、候補が多い場合も画面縦幅を占有しすぎない。
- SymbolDetail の CSV import は、既存の CSVテキスト貼り付けに加えて、CSVファイル選択から frontend でテキスト読込して既存 text-based API へ送信できる。
- 詳細な run / report 履歴は ApplicationDetail へ送る。

## 4. ApplicationDetail

- Symbol Strategy Application 単位の read-only detail。
- runs / reports history を filter / pagination 付きで表示する。
- run history は `run_type` / `run_status` で絞り込む。
- report history は `execution_source` / `run_type` / `status` / `with_metrics` で絞り込む。
- 実行系操作、CSV import、internal backtest execution、report conversion、削除は行わない。

## 5. Strategy 系画面

- StrategyList は保存済み strategy の一覧入口。
- StrategyDetail は strategy の概要、applied symbols、related reports を表示する。
- StrategyVersionDetail は version detail、Pine / validation / lineage / application 導線、strategy version 起点の CSV import / internal backtest 操作を表示する。
- StrategyLab は strategy 生成・保存・検証の作業画面であり、form framework は導入しない。
- LLM strategy proposal の初回実装では、StrategyLab に候補提案の入口を置き、deterministic stub provider から取得した候補を表示する。候補選択時は title と natural language spec へ反映し、proposal 選択だけでは Strategy / StrategyVersion 保存や Pine 生成へ自動連鎖しない。
- Proposal history / selected proposal lineage の初回 UI は StrategyLab が担当する。表示は「最近の提案」程度に留め、run status、created_at、provider、candidate count、selected 有無、候補 detail を最小表示する。
- StrategyLab は初回表示時と proposal 生成後に recent runs を取得する。過剰 polling、filter、pagination、bulk delete、retention UI は行わない。
- 現在表示中 proposal candidates と history detail candidates の「この候補を使う」は、履歴 selection API を呼んだうえで title / natural language spec へ反映する。selection API 失敗時は error 表示に留め、Pine generation / save / validation / backtest / AI summary は起動しない。
- 履歴候補を選択した場合も、title / natural language spec への反映だけを行う。Strategy / StrategyVersion 保存、Pine generation、validation、backtest は既存 button / form 操作を維持する。
- Codex CLI manual JSON import の初回 UI は StrategyLab が担当する。Codex CLI 用 prompt 作成、prompt 表示 / copy、JSON paste、任意の file text 読み取り、import 実行、import 成功後の candidate card 表示を最小 section として扱う。
- Codex CLI は画面から自動実行しない。ユーザーが手動で生成した `strategy_proposal_candidates` JSON を貼り付ける、または file picker で text として読み込む。
- import 成功後の candidate selection は既存 proposal と同じく title / natural language spec への反映だけを行い、Pine generation / save / validation / backtest / AI summary は自動起動しない。
- import error は sanitized reason だけを表示し、raw JSON、raw prompt、endpoint、model 実値、local path、stack trace を表示しない。
- StrategyVersionDetail に proposal lineage を表示する場合は後続判断とする。初回 history 実装では StrategyVersion への created-from relation を必須にしない。

## 6. Backtest 系画面

- BacktestList は report 一覧を表示する。
- BacktestDetail は個別 report detail、同一 application 関連 report、metrics comparison helper、AI summary、artifact を表示する。
- CSV import report と internal backtest report の metric 欠損は source に応じて短く説明する。
- BacktestComparisonDetail は保存済み comparison の再訪画面として扱い、本格 CSV / internal comparison route は後続判断とする。

## 7. 非対象

- ApplicationDetail への mutation 操作追加。
- metrics normalization を前提にした比較画面。
- Visual regression 本導入。
- SymbolDetail、Home、BacktestDetail、BacktestComparisonDetail、StrategyLab、StrategyVersionDetail などを対象にした visual snapshot。現行 pilot 対象は ApplicationDetail の application summary stable container 1 件のみ。
