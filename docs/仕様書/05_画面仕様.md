# 北極星 画面仕様 現行仕様

更新日: 2026-05-24
分類: 仕様書

## 1. 目的

本資料は、主要画面の現行責務を仕様として整理する。画面導線全体は `docs/仕様書/04_画面導線_IA.md` を参照する。

## 2. Home / SideRail

- Home は日次確認、summary、主要導線の入口。
- Home 本体は compact daily workspace として、大型の説明 section を置かず、マーケット概況 / AIデイリーサマリー / 最新アラート / 注目イベントを優先表示する。
- SideRail は監視銘柄 / 保有銘柄の主要導線と CRUD modal を担う。
- SideRail の折りたたみ状態は AppLayout 側の grid 幅にも反映し、折りたたみ時は main content が広がる。
- SideRail は viewport 高さ内で独立スクロールでき、長い監視 / 保有リストでもページ全体の最下部まで移動せずに下部を確認できる。
- SideRail の監視 / 保有一覧は compact list として扱い、表示見出しや補助説明を増やしすぎず、追加 / 編集 / 削除は短い button / icon button で表示密度を保つ。
- Watchlist / Positions の旧管理画面 route は移行期補助として扱い、主要導線は SideRail に寄せる。

## 3. SymbolDetail

- symbol の概要、latest alert / AI論点 / related references を表示する。
- related references はユーザー操作起点の手動再取得ボタンを持つ。表示起点、自動定期取得、batch 起点の refresh は行わない。
- saved application の概要、latest run、latest report、CSV / internal latest pair を表示する。
- server-side filter を使い、application status / report presence / report source / latest run type / latest run status / strategy / strategy version を切り替える。
- application 作成、archive / restore、CSV import、internal backtest start、internal report conversion の操作導線を持つ。
- 既存 strategy 選択は active strategy の検索、件数制限、ページ送りを持ち、候補が多い場合も画面縦幅を占有しすぎない。
- SymbolDetail の CSV import は、既存の CSVテキスト貼り付けに加えて、CSVファイル選択から frontend でテキスト読込して既存 text-based API へ送信できる。
- 詳細な run / report 履歴は ApplicationDetail へ送る。

## 4. ApplicationDetail

- Symbol Strategy Application 単位の read-only detail。
- runs / reports history を filter / pagination 付きで表示する。
- run history は `run_type` / `run_status` で絞り込む。
- report history は `execution_source` / `run_type` / `status` / `with_metrics` で絞り込む。
- 実行系操作、CSV import、internal backtest execution、report conversion、削除は行わない。

## 5. Strategy 系画面

- StrategyList は保存済み strategy の一覧入口。
- StrategyDetail は strategy の概要、applied symbols、related reports を表示する。
- StrategyVersionDetail は version detail、Pine / validation / lineage / application 導線、strategy version 起点の CSV import / internal backtest 操作を表示する。
- StrategyLab は strategy 生成・保存・検証の作業画面であり、form framework は導入しない。
- StrategyLab の Pine generation / strategy proposal 入力は `market=JP_STOCK|US_STOCK`、canonical `timeframe=D|4H|1H` を選択肢として表示する。`D` は「日足（D）」、`4H` は「4時間足（4H）」、`1H` は「1時間足（1H）」として表示し、`1D` は UI 選択肢に出さず backend で `D` に正規化する。default は既存通り `JP_STOCK` / `D` とする。
- StrategyLab の strategy proposal section には `提案用時間足` selector を置き、proposal generation、Codex CLI prompt、Web検索付き prompt option は同じ selected timeframe を使う。時間足により提案候補の前提や注意点が変わることを helper text で示す。
- StrategyLab の strategy proposal section には `提案用ヒント（任意）` textarea を置く。これは Pine 生成用 natural language rule とは別入力であり、空欄時は `user_hint=null` として幅広い候補を要求する。候補選択時も proposal hint は自動上書きしない。
- StrategyLab と StrategyVersionDetail は、生成した Pine を TradingView の symbol / chart timeframe 上で検証する前提と、internal backtest の対応範囲拡張ではないことを helper text で示す。
- unsupported market / timeframe は backend の Pine generation warning / assumption で扱い、画面から自動で internal backtest や AI summary を起動しない。Pine 生成 provider には warnings / assumptions をユーザー向け日本語文で返すよう要求し、生成 script 本文や Pine 構文は翻訳しない。post-processing が追加する warning も日本語文にするが、provider 自由文の後段固定 mapping / 翻訳 provider は行わない。
- StrategyLab / StrategyVersionDetail の Pine generation は LLM-first として説明する。deterministic generator は baseline / fallback / test 用であり、通常の生成品質は LLM output、validation、bounded repair の結果として確認する。
- Pine generation の画面表示では `repair_attempts` / `invalid_reason_codes` / sanitized failure reason を確認できるが、raw prompt、raw provider response、endpoint、model 実値、secret、local path、stack trace は表示しない。
- TradingView compile は自動化しない。ユーザーが TradingView で確認した compile error や検証メモを、既存の Pine 修正再生成入力へ手動で戻す。
- LLM strategy proposal の初回実装では、StrategyLab に候補提案の入口を置き、deterministic stub provider から取得した候補を表示する。候補選択時は title と natural language spec へ反映し、proposal 選択だけでは Strategy / StrategyVersion 保存や Pine 生成へ自動連鎖しない。
- Proposal history full management は StrategyLab が担当する。既存の「最近の提案」を compact な「提案履歴」導線へ拡張し、provider / status / selected / search / pagination で保存済み run を探せる。
- 提案履歴 list は run status、created_at、provider、candidate count、selected 有無を最小表示する。candidate title / summary / suggested spec は detail 展開後に確認し、list では user_hint 全文、raw prompt、raw provider response、raw Codex output、endpoint、model 実値、local path、stack trace を表示しない。
- StrategyLab は proposal history soft archive UI を最小追加する。archived filter は `active` default、`archived`、`all` を持ち、filter 変更時は page を 1 に戻す。
- active run には `アーカイブ` action、archived run には `戻す` action を表示する。archive は削除ではなく通常一覧から隠す操作であることを helper text で明示する。
- archived run は badge で表示し、archived filter または all filter から detail を開ける。archived run の candidate selection は既存どおり title / natural language spec への反映だけを行い、自動 unarchive はしない。
- StrategyLab は初回表示時、proposal 生成後、Codex CLI import 後、candidate selection 後、archive / unarchive 後に proposal history を取得する。過剰 polling、bulk delete、hard delete、retention、export UI は行わない。
- 現在表示中 proposal candidates と history detail candidates の「この候補を使う」は、履歴 selection API を呼んだうえで title / natural language spec へ反映する。selection API 失敗時は error 表示に留め、Pine generation / save / validation / backtest / AI summary は起動しない。
- 履歴候補を選択した場合も、title / natural language spec への反映だけを行う。Strategy / StrategyVersion 保存、Pine generation、validation、backtest は既存 button / form 操作を維持する。
- Codex CLI manual JSON import の初回 UI は StrategyLab が担当する。Codex CLI 用 prompt 作成、prompt 表示 / copy、JSON paste、任意の file text 読み取り、import 実行、import 成功後の candidate card 表示を最小 section として扱う。
- Codex CLI は画面から自動実行しない。ユーザーが手動で生成した `strategy_proposal_candidates` JSON を貼り付ける、または file picker で text として読み込む。
- Codex CLI 用 prompt 作成には「Codex CLI側でWeb検索を使う前提のpromptにする」checkbox を置く。checked の場合だけ prompt に Web 検索利用時の注意文を追加するが、北極星は Web 検索を自動実行せず、import 時にも Web 検索済みかどうかを判定しない。
- Web検索付き prompt option は UI 上の prompt 文面切替に限定し、JSON import、candidate cards、selection flow、Pine generation / save / backtest の手動導線は既存通り維持する。
- import 成功後の candidate selection は既存 proposal と同じく title / natural language spec への反映だけを行い、Pine generation / save / validation / backtest / AI summary は自動起動しない。
- import error は sanitized reason だけを表示し、raw JSON、raw prompt、endpoint、model 実値、local path、stack trace を表示しない。
- StrategyVersionDetail に proposal lineage を表示する場合は後続判断とする。初回 history 実装では StrategyVersion への created-from relation を必須にしない。

## 6. Backtest 系画面

- BacktestList は report 一覧を表示する。
- BacktestDetail は個別 report detail、同一 application 関連 report、metrics comparison helper、AI summary、artifact を表示する。
- CSV import report と internal backtest report の metric 欠損は source に応じて短く説明する。
- BacktestComparisonDetail は保存済み comparison の再訪画面として扱い、本格 CSV / internal comparison route は後続判断とする。

## 7. 非対象

- ApplicationDetail への mutation 操作追加。
- metrics normalization を前提にした比較画面。
- Visual regression 本導入。
- SymbolDetail、Home、BacktestDetail、BacktestComparisonDetail、StrategyLab、StrategyVersionDetail などを対象にした visual snapshot。現行 pilot 対象は ApplicationDetail の application summary stable container 1 件のみ。

## 8. UI/UX production readiness phase 1

- UI/UX production readiness phase 1 は、既存機能を増やさず、AppLayout / Navigation / SideRail / Home / SymbolDetail / StrategyLab の情報階層、余白、section grouping、操作可能箇所の視認性を改善する。
- AppLayout は desktop web app 前提の app shell とし、header、SideRail、main content の密度と背景を揃える。
- Navigation は主要 route への入口を維持し、現行 route や IA は変更しない。
- SideRail は監視銘柄 / 保有銘柄の主要導線として、row のクリック可能箇所、編集 / 削除 action、補助説明の視認性を上げる。
- Home は daily workspace として、SideRail へ送る情報と Home 本体で見る情報を分け、マーケット概況、AIデイリーサマリー、最新アラート、注目イベントの section を読みやすくする。
- SymbolDetail は loading / filter / saved application 周辺の視認性を上げるが、CSV import、internal backtest、application archive / restore、references refresh の挙動は変えない。
- StrategyLab は proposal / Codex CLI import / proposal history / rule input / Pine generation / CSV import の作業単位を section で分け、candidate selection が title / natural language spec 反映に留まる既存境界を維持する。
- Phase 1 では API shape、backend business logic、DB / Prisma schema、DataTable 高機能化、modal framework 全面導入、visual regression required check 化は行わない。

## 9. UI/UX production readiness phase 2

- UI/UX production readiness phase 2 は、ApplicationDetail / BacktestDetail / StrategyDetail / StrategyVersionDetail / BacktestComparisonDetail を対象に、検証・履歴系画面の外枠、見出し、section density、loading / error / empty state の見え方を改善する。
- ApplicationDetail は run / report 履歴の filter、card、linked metadata の密度を揃える。ApplicationDetail から mutation 操作、CSV import、internal backtest execution、report conversion は追加しない。
- BacktestDetail は AppLayout / PageHeader を使い、基本情報、使用 Strategy、AI summary、import 履歴へ進む流れを明確にする。AI summary auto enqueue、manual generate、comparison helper、artifact metadata の既存責務は変更しない。
- StrategyDetail は strategy 概要、version 一覧入口、applied symbols、related reports の section 境界を揃える。archive / restore は既存 status 操作として維持し、favorite / hard delete は追加しない。
- StrategyVersionDetail は rule version の基本情報、自然言語 rule、Pine、validation、internal backtest 導線を同じ page shell に載せる。Pine generation、validation、save、internal backtest は既存 button / form 操作を維持する。
- StrategyVersionDetail の Pine 再生成は LLM output を validation / bounded repair に通すが、TradingView compile automation、auto save、auto backtest、AI summary 自動生成は追加しない。
- BacktestComparisonDetail は保存済み pairwise comparison の再訪画面として AppLayout / PageHeader / SectionCard に揃える。本格比較画面、AI summary 同士の自動比較、artifact diff は後続判断とする。
- Phase 2 では API shape、backend business logic、DB / Prisma schema、DataTable 高機能化、modal framework 全面導入、visual regression required check 化は行わない。

## 10. UI/UX production readiness phase 3

- UI/UX production readiness phase 3 は、Phase 1 / Phase 2 で重複した filter block、history card、proposal card、nested metadata、read-only text block の surface 表現を最小共通化する。
- `Surface` component は Home / SymbolDetail / StrategyLab / ApplicationDetail / BacktestComparisonDetail の繰り返し表現を整理する用途に限定する。
- 画面ごとの責務、文言、API 呼び出し、mutation、proposal selection、Pine generation、backtest、AI summary の挙動は各画面に残す。
- Phase 3 では DataList / SimpleTable / DataTable、高機能 table、virtual scroll、column resize、modal framework 全面導入、visual regression required check 化は行わない。

## 11. Core screens compact density pass

- Core screens compact density pass は、SymbolDetail / ApplicationDetail / BacktestDetail / StrategyLab / StrategyVersionDetail / StrategyDetail / BacktestComparisonDetail / StrategyList / BacktestList の余白、section density、list card padding を小さく整える範囲に限定する。
- PageHeader / SectionCard / Surface の既定余白を一段詰め、画面固有の API 呼び出し、mutation、refresh、provider、job enqueue、proposal selection の責務は変更しない。
- Proposal から Pine / save / backtest / AI summary への自動連鎖、DataTable 高機能化、virtual scroll、column resize、modal framework 全面導入、Visual regression required check 化は行わない。
