# 北極星 MVPタスク分解

更新日: 2026-03-15

## 1. このドキュメントの目的

本ドキュメントは、株価評価AIツール「北極星」のMVP開発に向けて、これまで整理してきた企画・仕様・設計内容をもとに、**実装可能なタスク単位** へ分解することを目的とする。

北極星は、以下の3コンセプトを実現するために設計されている。

- 毎日5分で状況把握
    
- 銘柄研究の高速化
    
- ルール運用の検証
    

MVPでは、これら3つを最小限成立させることを目的とし、機能を広げすぎず、**依存関係を踏まえて順序立てて実装する** ことを重視する。

---

## 2. タスク分解の基本方針

### 2.1 ユーザー価値よりも先に基盤を置く

MVPでは最終画面を早く見たいが、北極星は

- webhook受信
    
- AI要約
    
- 自然言語→Pine
    
- CSV取込  
    といった基盤依存が強い。
    

そのため、

- データモデル
    
- API
    
- 外部連携
    
- 非同期処理  
    を先に成立させる必要がある。
    

### 2.2 機能単位ではなく「縦の流れ」で優先する

以下の縦導線を重視する。

1. TradingView alert → 北極星受信 → AI要約 → ホーム表示
    
2. 銘柄詳細 → references → AI論点 → 研究ノート
    
3. 自然言語ルール → normalized_rule_json → Pine → backtest run → CSV取込 → 総評
    

### 2.3 MVPでは作らないものを明示する

タスク分解では、後回しにするものも明示してスコープを守る。

## 17. 結論

北極星MVPのタスク分解では、機能を単純に横並びで作るのではなく、  
**TradingView alert → 要約 → ホーム**、  
**銘柄詳細 → 論点 → ノート**、  
**自然言語 → Pine → CSV取込 → 比較**  
という 3 本の縦導線を成立させることを優先する。

そのため、MVPでは以下を重視する。

- まず基盤を置く
    
- webhook と AI要約を先に動かす
    
- 銘柄研究機能を次に成立させる
    
- その後にルール検証と CSV取込へ進む
    
- 比較と総評はその上に乗せる
    

この順序で進めることで、北極星のMVPを過不足なく実装しやすい形に落とし込むことができる。
