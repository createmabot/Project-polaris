## 5. 再発確認

### content 空 / reasoning のみ / length 打ち切り

- Symbol / Comparison / Backtest / Home の今回観測では再発なし。
- local LLM の `finish_reason=length` は今回の実測では未発生。
- provider エラー全文や secret 類の画面露出も未観測。

### references 不足時の期待表現

- `reference_count = 0` は `insufficient_context = true` と扱う。
- 画面上では「参照情報が不足しており、snapshot / note 中心の仮説」であることを明示する。
- 「情報が一切ない」とは表現しない。

## 6. 今回の調整

### 実装調整

- `backend/src/ai/home-provider.ts`
  - local LLM 用の system prompt を、断定的な売買推奨を避ける内容へ調整。
  - `reference_count = 0` の Symbol summary で、body markdown に参照不足を明示。
  - Comparison summary で、片側 `recent_reference_count = 0` の場合は `confidence = medium` に抑制。
  - Symbol summary の `bullish_points / bearish_points / next_events` に含まれる `reference_ids` を、実在する `referenceIds` のみへ正規化。
  - deterministic の日次 / symbol / comparison 補助文言を UTF-8 正常文へ更新。

### 観測上の改善結果

- 6758 の論点カード:
  - 修正前: 参照 0 件でも body が曖昧で、overall view がやや強気寄り。
  - 修正後: body に「参照情報が不足しており、snapshot / note 中心の仮説です」を表示。
- 7203 vs 6758 比較総評:
  - 修正前: 6758 側 references 0 件でも `confidence=high`。
  - 修正後: 片側 references 不足を考慮し `confidence=medium`。

## 8. 残課題

- references 0 件でも、local LLM は業界一般論ベースの説明を出しがちである。
  - 画面・body では不足を明示したが、本文自体の保守性は今後さらに詰める余地がある。
- Comparison summary は `insufficient_context=false` のままでも、片側不足のケースは存在する。
  - 今回は `confidence` を下げるまでに留めた。
- Backtest summary は入力ハッシュが同一の場合、既存 summary を再利用する。
  - 同条件再生成で prompt 差分だけを比較したい場合は別タスクが必要。
