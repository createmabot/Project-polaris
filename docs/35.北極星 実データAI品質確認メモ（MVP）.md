### 10.7 今回の実装調整

- `backend/src/ai/local-llm-adapter.ts`
  - Alert summary 用 system prompt を、買い/売り/警戒/監視シグナルとして評論的に評価する方針へ更新。
  - 無条件・保証的な断定を避けつつ、signal evaluation を避けないように調整。
  - `/v1/chat/completions` 失敗時に `/api/chat` へフォールバック。
  - `reason_hypotheses.reference_ids` を実在する `reference_ids` のみに正規化。
  - body markdown を `何が発火したか / シグナル評価 / 背景材料 / 追加確認` の構造に揃えた。
- `backend/src/ai/context-builder.ts`
  - `alert_event` 直結 refs が 0 件でも、同一 symbol の既存 refs を補助的に参照できるよう調整。
  - symbol fallback references は alert 発火時刻以前に限定し、look-ahead bias を回避。

### 10.8 品質評価方針の更新

- 旧方針:
  - 「買うべき / 売るべき のような直接的投資助言がない」ことを主に良い点として評価していた。
- 新方針:
  - Alert summary では、アラートを買い/売り/警戒/監視シグナルとして評価する。
  - ただし、必ず買うべき、絶対売るべき、確実に上がる、といった保証的表現は避ける。
  - references が不足している場合も signal evaluation 自体は残し、「背景補強が弱い」と明示する。
  - references が順風なら補強材料、逆風なら逆風材料として扱う。

### 10.9 次タスク候補

1. dedupe 済み existing refs を alert_event 側へどう見せるかを、`context-builder` 補助参照の方針で継続整理する。
2. Alert summary の `reason_hypotheses` に、技術的条件の具体性が足りない場合の定型文を少し詰める。
