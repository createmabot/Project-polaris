# 北極星 ローカル開発環境

更新日: 2026-05-13
分類: 運用ドキュメント

## 1. 目的

本資料は、北極星をローカルで開発・確認するための環境準備手順をまとめる。詳細な履歴は `README.md` と `docs/24.北極星 開発着手用 README セットアップ手順書（MVP）.md` を参照し、本資料では日常運用で使う最短手順を扱う。

## 2. 前提

- Node.js 22 を使う。
- package manager は pnpm workspaces を使う。
- PostgreSQL と Redis は Docker Compose で起動する。
- backend は Fastify + TypeScript、frontend は Vite + React + TypeScript で起動する。
- DB 定義は Prisma schema と migration を正本にする。

## 3. 初回セットアップ

1. 依存関係をインストールする。

```bash
pnpm install
```

2. 環境変数ファイルを準備する。

```bash
cp .env.example .env
```

3. PostgreSQL / Redis を起動する。

```bash
pnpm run up
```

または Docker Compose を直接使う。

```bash
docker compose up -d
```

4. DB を反映する。

```bash
cd backend
pnpm exec prisma validate
pnpm exec prisma migrate deploy
pnpm exec prisma generate
pnpm exec prisma db seed
```

5. repository root に戻り、開発サーバーを起動する。

```bash
cd ..
pnpm run dev
```

## 4. 日常の起動順

通常は次の順で十分である。

1. `pnpm run up`
2. `cd backend && pnpm exec prisma migrate deploy && pnpm exec prisma generate && pnpm exec prisma db seed`
3. `cd .. && pnpm run dev`
4. ブラウザで frontend を開く
5. `docs/walkthrough.md` の確認導線を必要な範囲だけ実施する

## 5. DB refresh の扱い

Windows で DB の再起動、migration、seed をまとめて実行したい場合は `db-refresh.bat` を使える。

```bat
.\db-refresh.bat
```

注意:

- 実行前に backend / frontend の dev process を止める。
- Prisma engine の file lock が残る場合は、関連 process を停止してから再実行する。
- production 相当のデータを local に持ち込まない。

## 6. 環境変数の扱い

- `.env.example` を雛形として使う。
- secret / token / API key の実値を docs、PR、issue、log に残さない。
- `HOME_AI_PROVIDER` は `stub|local_llm|openai_api` のいずれかで切り替える。
- `AI_ENABLE_STUB_FALLBACK` は必要な検証時だけ明示する。既定では provider failure を failed job として扱う。
- TradingView webhook token や shared secret は実値を書かず、placeholder で表現する。

## 7. 文字コード

- docs は UTF-8 日本語で保存する。
- PowerShell から日本語 JSON を送る場合は、`docs/walkthrough.md` の UTF-8 byte array 手順を使う。
- 表示上の文字化けとファイル実体の文字化けは分けて確認する。
