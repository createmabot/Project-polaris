# 北極星 テストと CI

更新日: 2026-05-13
分類: 運用ドキュメント

## 1. 目的

本資料は、北極星で使う test / CI の日常運用をまとめる。仕様としてのテスト方針は `docs/仕様書/10_テスト仕様.md`、browser-based E2E の背景は `docs/45.北極星 browser-based E2E導入方針（P3）.md` を参照する。

## 2. テストの役割分担

- backend API E2E: API 契約、保存導線、queue / collector / summary flow を固定する。
- frontend Vitest: component 文言、空状態、URL helper、表示分岐を固定する。
- frontend flow test: return-flow や URL query の契約を固定する。
- Playwright browser smoke: 実ブラウザで主要 read-only 導線が成立することを確認する。
- manual walkthrough: TradingView、CSV file 選択、外部ツール連携など、自動化しすぎない確認を担う。

## 3. よく使うコマンド

```bash
pnpm run build
pnpm run test
pnpm --filter backend test:e2e:home-symbol-comparison
pnpm --filter frontend test:e2e:browser
pnpm test:e2e:browser
git diff --check
```

作業範囲が小さい場合は、該当 package / spec に絞って実行する。docs-only PR では build / test を無理に実行せず、`git diff --check` と docs scan を優先する。

## 4. Browser smoke の前提

Playwright smoke は local-only から始める。

前提:

- PostgreSQL / Redis 起動済み
- backend 起動済み
- frontend 起動済み
- migration / seed 済み
- 外部 API / LLM / TradingView に依存しない導線を優先する

初回 browser install が必要な場合だけ次を実行する。

```bash
pnpm --filter frontend exec playwright install chromium
```

## 5. CI required checks

GitHub Actions では次の checks を運用する。

- `strategy-versions-return-flow-e2e-check`
- `backtests-return-flow-e2e-check`
- `snapshot-review-generator-json-check`
- `symbol-snapshot-db-integration`

`main` の required checks は ruleset で管理する。check 名と workflow 名がずれると `Expected - Waiting for status to be reported` になるため、ruleset と実際の check 名を合わせる。

## 6. Required-check failure drill

required check failure 時に merge が block されることを定期確認する。四半期に 1 回、または ruleset / branch protection 変更時に実施する。

最小手順:

1. `main` から検証ブランチを作成する。
2. 対象 check が落ちる最小の一時破壊を入れる。
3. PR を作成し、対象 check が failure になることを確認する。
4. PR が required check 未通過で block されることを確認する。
5. 一時破壊を revert し、success に戻ることを確認する。
6. 結果を snapshot weekly review の運用記録へ残す。

検証用の破壊コミットは `main` に merge しない。

## 7. docs-only PR の確認

- `git diff --check` を実行する。
- 変更 docs に local path、secret、token、shared_secret、API key の実値がないことを scan する。
- 実装コード、API、backend、frontend、DB、Prisma schema、tests を変更していないことを確認する。
- PR 本文は ASCII English only にする。
