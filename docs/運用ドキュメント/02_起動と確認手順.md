# 北極星 起動と確認手順

更新日: 2026-05-13
分類: 運用ドキュメント

## 1. 目的

本資料は、ローカル起動後に主要導線が成立しているかを確認するための手順をまとめる。詳細な受け入れ確認は `docs/walkthrough.md` を正本とし、本資料は日常のクイック確認に限定する。

## 2. 起動手順

1. PostgreSQL / Redis を起動する。

```bash
pnpm run up
```

2. DB を最新化し、seed を投入する。

```bash
cd backend
pnpm exec prisma validate
pnpm exec prisma migrate deploy
pnpm exec prisma generate
pnpm exec prisma db seed
```

3. repository root へ戻って開発サーバーを起動する。

```bash
cd ..
pnpm run dev
```

## 3. 最小画面確認

seed 後は、次の順に確認する。

1. `/home` または `/` を開き、Home の主要 block が表示されることを確認する。
2. SideRail の watchlist または positions から SymbolDetail へ遷移する。
3. SymbolDetail で current snapshot、関連参照情報、recent alerts、保存済み application の表示を確認する。
4. `/strategies` または StrategyDetail で strategy / version の read-only 情報を確認する。
5. `/backtests` から BacktestDetail へ遷移し、CSV import report または internal backtest report の表示を確認する。
6. 必要に応じて `/symbol-strategy-applications/:applicationId` で run / report history を確認する。

## 4. Rule Lab / Backtest 一巡確認

実装変更や CSV / Pine / AI summary 周辺を触った場合は、`docs/walkthrough.md` の Rule Lab / Backtest 一巡を使う。

最小観点:

1. strategy 作成
2. strategy version 作成
3. Pine 生成
4. TradingView 一次検証メモの反映
5. CSV import
6. Backtest Detail 表示
7. Backtest AI summary 表示または生成
8. inline comparison / saved pairwise comparison

## 5. API quick check

必要な場合だけ、代表 API を確認する。

- `GET /api/home`
- `GET /api/symbols/:symbolId`
- `GET /api/backtests/:backtestId`
- `GET /api/symbols/:symbolId/strategy-applications`
- `GET /api/symbol-strategy-applications/:applicationId/runs`
- `GET /api/symbol-strategy-applications/:applicationId/reports`

API shape の仕様判断は `docs/仕様書/03_API仕様.md` と機能別正本 docs を優先する。

## 6. 終了手順

1. dev server を停止する。
2. DB / Redis を止める。

```bash
pnpm run down
```

## 7. 確認結果の残し方

- 通常の作業 PR では、実施した build / test / manual check を PR 本文に ASCII English で記録する。
- docs-only PR では `git diff --check` と secret/local path scan の結果を記録する。
- 手順の追加や運用判断が生じた場合は、該当する `docs/運用ドキュメント/` または `docs/作業進捗管理/` に追記する。
