## 6. PR #337〜#340 大フェーズ完了

PR #337〜#340 は、artifact / AI summary / visual regression の後続整理を、実装済み境界と見送り範囲を分けて完了扱いにする。

## 9. Strategy proposal provider expansion 完了判断

Strategy proposal provider expansion は、PR #352〜#353 の docs design と boundary implementation をもって、non-stub provider 実装前の境界固定フェーズとして完了扱いにする。

完了範囲:

- Strategy proposal provider boundary を固定した。
- deterministic stub provider を provider boundary 配下に整理した。
- request validation と provider response validation を導入した。
- `user_hint` の長文を bounding し、proposal へ過剰に混入しない扱いを整理した。
- `user_hint` 由来の投資助言風表現は request `VALIDATION_ERROR` にしない。入力として許容し、type validation と length bounding は維持する。
- provider output 由来の schema / type / format / candidate count 不正は `PROVIDER_INVALID_RESPONSE` として扱う。単なる投資助言風 wording だけでは reject しない。
- 安全のために検証候補を狭めすぎず、UI / docs の disclaimer と backtest / user review required の前提で扱う。
- empty candidates は StrategyLab の EmptyState で扱える成功応答として許容する。
- `strategy_type_bias=other` は empty candidates になる既存挙動を維持する。

完了扱いに含めないもの:

- openai_api provider。
- request-time provider selection。
- local_llm failure 時の opt-in fallback metadata。
- Web search / deep research。
- proposal history DB persistence。
- auto Pine generation / auto save。
