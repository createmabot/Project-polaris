# 北極星 Rule Lab日本語入力 UI経路再確認メモ（MVP後P1）

## 1. 目的

- Rule Lab の日本語自然文入力について、UI経路と shell / PowerShell 経路を分けて再確認する。
- UI経路の通常利用可否を記録する。
- PowerShell 文字化け制約の扱いを明確化する。

## 2. 背景

- MVP受入では Rule Lab / Pine は `OK`。
- PowerShell 経由では encoding 由来の文字化け失敗がありうる。
- backend には mojibake detection と `backend/test/japanese-input.e2e.test.ts` がある。
- UI経路ではブラウザから JSON 送信されるため、通常は UTF-8 として扱える。

## 3. 確認対象導線

- `/strategy-lab`
- strategy 作成
- strategy version 作成
- Pine 生成
- generated pine 表示
- version detail 表示
- Pine 修正再生成は補足確認

## 4. UI経路の確認結果

| 確認項目 | 結果 | 備考 |
|---|---|---|
| 日本語自然文入力 | OK | in-app browser 上で textarea に既定の日本語自然文が入り、通常表示された |
| strategy作成 | OK | `/strategy-lab` から保存して生成を実行し、`strategy_id` が払い出された |
| version作成 | OK | `version_id` が払い出され、version 一覧 / 詳細導線が表示された |
| 日本語保存 | OK | version detail で `natural_language_rule` を日本語のまま再表示できた |
| Pine生成 | OK | UI操作で generated status になり、Pine が表示された |
| generated pine表示 | OK | StrategyLab / StrategyVersionDetail の両方で generated pine を確認した |
| warnings表示 | 要確認 | 正常な日本語入力では warning なし。文字化け疑い warning は backend API / e2e で確認済みだが、StrategyLab UI に `creation_warnings` の専用表示はない |
| version detail再表示 | OK | version detail で日本語自然文、Pine 状態、generated pine を再確認した |

## 5. 実ブラウザ確認メモ

- 確認方法: in-app browser で `http://localhost:5173/strategy-lab` を開いて一巡確認
- 使用した日本語自然文:
  - `25日移動平均線の上で、RSIが50以上、出来高が20日平均の1.5倍以上で買い。終値が5日線を下回ったら手仕舞い。`
- 確認できたこと:
  - StrategyLab の textarea に日本語が通常表示される
  - `保存してPine生成` 実行後に `strategy_id` / `version_id` / `status: generated` / `backtest_id` が表示される
  - `generated pine` が表示される
  - `この version 詳細を開く` から詳細画面に遷移し、日本語自然文を再確認できる
- 補足:
  - assumptions は英語文で返るケースがあった
  - これは日本語入力の破綻ではなく、Pine 生成側の出力品質の範囲であり、本タスクでは blocker としない

## 6. shell / PowerShell経路の扱い

- PowerShell 経路は通常利用経路ではない。
- 日本語を送る場合は UTF-8 byte array + `charset=utf-8` を使う。
- 文字化け疑いは warning として扱う。
- 自動修復はしない。
- UI経路の失敗と shell encoding 問題を混同しない。

## 7. 受入条件

- UIから日本語自然文を入力して version に保存できる。
- 正常な日本語では文字化け warning が出ない。
- Pine生成の成否を、日本語入力問題と provider / Pine生成問題に切り分けられる。
- version detail で日本語自然文を再確認できる。
- PowerShell 経路は runbook の制約として記録されている。

## 8. 既存テストで確認できること

- `backend/test/japanese-input.e2e.test.ts`
  - 正常な UTF-8 日本語 `natural_language_rule` が保存できる
  - 正常な日本語では `creation_warnings` が空
  - `REPLACEMENT CHARACTER` を含む文字列では warning を返す
- `frontend/src/pages/StrategyVersionDetail.test.tsx`
  - 日本語 `natural_language_rule` を含む detail 表示データを扱っている
- `frontend/src/pages/StrategyLab.test.tsx`
  - StrategyLab 初期表示で日本語自然文が保持されることを固定

## 9. 残課題

- Browser based E2E は未導入のため、今回の UI live 確認は手動確認メモとして残す。
- shell 経路の完全救済は対象外。
- docs 全体の文字化け棚卸しは別 P1 タスク。
- Pine 生成品質改善や assumptions の日本語統一は別タスク。
