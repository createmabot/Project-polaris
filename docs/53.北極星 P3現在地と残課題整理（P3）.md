# 北極星 P3現在地と残課題整理（P3）

## 1. 目的

この文書は、PR #247 時点の P3 現在地と残課題を整理するためのメモである。
既存の詳細設計は docs/46 / docs/47 / docs/49 / docs/52 を正本とし、この文書では完了扱い、残課題、後続フェーズ送りを機能領域ごとにまとめる。

## 2. P3で完了扱いにするもの

### SideRail / Home / AppLayout

- `AppLayout` / `PageHeader` / `Navigation` / `TextLink` の最小土台を追加済み。
- `Home` は日次確認ダッシュボード寄りに整理済みで、監視 / 保有の詳細一覧は `SideRail` 側へ寄せた。
- `SideRail` の watchlist / positions CRUD モーダル化は完了済み。
- `Home` と `SideRail` の `/api/home?summary_type=latest` 二重 fetch は整理済み。
- CRUD 後の refresh 範囲は watchlist / positions 系ごとに分け、不要な管理 API 再取得を減らした。

### StrategyList / StrategyDetail

- `/strategies` と `/strategies/:strategyId` の placeholder route を追加済み。
- 既存 `StrategyRule` / `StrategyRuleVersion` の read-only 表示を追加済み。
- `StrategyRule.status` を `active` / `archived` として扱う方針を docs/51 に固定し、StrategyList / StrategyDetail から archive / restore できるようにした。
- `StrategyDetail` では、適用済み銘柄と関連検証レポートを read-only で確認できる。

### Symbol Strategy Application

- docs/52 で Symbol Strategy Application / Application Run / Backtest Report の関係を正本化済み。
- Prisma schema / migration に `SymbolStrategyApplication` と `SymbolStrategyApplicationRun` を追加済み。
- `GET /api/symbols/:symbolId/strategy-applications` と `POST /api/symbols/:symbolId/strategy-applications` を追加済み。
- `SymbolDetail` から active strategy / version を選び、Symbol Strategy Application として保存できる。
- application archive / restore API と最小 UI 接続を追加済み。
- hard delete は実装していない。

### CSV import wiring

- 保存済み Symbol Strategy Application 起点で TradingView CSV import を実行できる。
- CSV import 時に `Backtest` / `BacktestImport` / `SymbolStrategyApplicationRun` を作成し、run から report へ接続する。
- parse failed の場合も run / import として保存し、UI では成功表示と区別する。

### internal backtest wiring

- 保存済み Symbol Strategy Application 起点で internal backtest execution を開始できる。
- `InternalBacktestExecution` と `SymbolStrategyApplicationRun` を作成し、run から execution へ接続する。
- application 起点では application の symbol を強制的に使用するようにし、別銘柄 execution が application 配下に記録されないようにした。
- `SymbolDetail` で latest run が `internal_backtest` の場合、execution status / result summary を read-only で確認できる。
- Backtest report 化はまだ行っていない。

### SymbolDetail 表示

- `SymbolDetail` の主要 section は最小 Tailwind 化済み。
- `ストラテジー / 検証結果` section を追加し、strategy / version 選択、application 保存、保存済み application 表示、CSV import、internal backtest 起動、latest run / latest report 表示まで接続済み。
- 表示肥大化に対して、保存済み application / run / report 表示を `SymbolDetail` 内 private component に分割済み。

### BacktestDetail backlink

- `BacktestDetail` は個別検証レポート詳細として維持している。
- Symbol Strategy Application 由来の report では、`銘柄起点の適用情報` として application / run / symbol / strategy / strategy version の backlink を表示する。
- BacktestDetail の backlink 表示は private component に分割し、表示内容と導線を維持したまま読みやすく整理した。

### Playwright smoke

- Home → SymbolDetail の既存 smoke は維持している。
- P3 の代表的な read-only 導線として、Home → SymbolDetail → StrategyDetail → BacktestDetail の browser smoke を追加済み。
- CRUD / CSV import 実行 / internal backtest 実行 / archive restore 実行は smoke には含めていない。

### UI component cleanup

- `SectionCard` を追加し、Home / SymbolDetail の一部 section wrapper へ限定適用した。
- SymbolDetail / BacktestDetail の肥大化した表示は、画面内 private component へ小さく分割した。
- Button / EmptyState / LoadingState の共通化や全面リデザインは行っていない。

- SideRail の状態永続化や API 最適化の追加検討。

- Playwright scenario の追加。ただし CRUD / import / execution を含むものは fixture と安定性を確認してから行う。
