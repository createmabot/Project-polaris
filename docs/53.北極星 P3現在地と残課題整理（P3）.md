## 追記（2026-05-11）CSV import report / internal backtest report 比較 UX 方針

### 役割差

- CSV import report は TradingView 実行結果を保存する report とする。`BacktestImport` と parsed summary を持ち、外部ツールで実行した実運用に近い検証結果として扱う。

- どちらも `BacktestDetail` で個別検証レポートとして表示するが、由来と入力文脈は明確に分ける。

### 比較したい項目

- `execution_source`、market / timeframe、strategy version、period、metrics、trade count、return、drawdown、artifact pointer、AI summary を比較対象の中心にする。
- CSV import 由来では parsed summary を主な根拠にし、internal backtest 由来では `strategySnapshotJson.result_summary` と `artifact_pointer` を主な根拠にする。
- raw JSON 全体比較ではなく、まずは report の由来、期間、主要 metrics、AI summary の差分が読み取れる UX を優先する。

- `BacktestList` / comparison route: 本格比較画面は後続フェーズで検討する。

### latest_run 基準

- `run_type` / `run_status` は application の最新 run に対する filter として扱う。
- `latest_run.run_type` が `csv_import` の application、または `internal_backtest` の applicationを絞り込む。

- application 配下のどれかの run が条件に合う application を返す案。
