# 北極星 TradingView CSV import 運用手順（MVP）

更新日: 2026-05-05

## 1. 目的

本ドキュメントは、TradingView の Strategy Report / Strategy Tester から出力した CSV を、
北極星の Rule Lab / Backtest へ取り込む実運用手順を固定するための runbook である。

役割分担は維持する。

- TradingView: 表示 / 一次検証 / CSV 出力
- 北極星: CSV 保存 / parse / 履歴管理 / AI 総評 / 比較

## 2. 前提

- Pine Script は TradingView 上で一次検証する
- 北極星では `POST /api/backtests/:backtestId/imports` で CSV を JSON body の `csv_text` として受け取る
- 画面上の導線は `/strategy-lab` と `/backtests/:backtestId`
- 実装上の bodyLimit は 2 MB
- 対応している CSV は以下
  - Performance Summary（英語ヘッダー）
  - List of Trades（英語ヘッダー）
  - List of Trades（日本語ヘッダー）

## 3. TradingView 側で CSV を出す手順

### 3.1 Pine Script を開く

1. TradingView で対象チャートを開く
2. Pine Script を貼り付ける
3. Strategy として実行する

### 3.2 Strategy Report / Strategy Tester を開く

- TradingView の UI では `Strategy Report` と `Strategy Tester` の表記揺れがある
- 北極星ではどちらの表記でも、バックテスト結果の CSV を出す画面として扱う

### 3.3 Performance Summary を export する場合

1. `Performance Summary` タブを開く
2. CSV export 操作を行う
3. ファイル名は任意だが、運用上は `performance_summary_*.csv` を推奨する

用途:

- 主要指標をそのまま北極星へ入れたい場合
- `Net Profit / Total Closed Trades / Percent Profitable / Profit Factor / Max Drawdown / From / To`
  を直接確認したい場合

### 3.4 List of Trades を export する場合

1. `List of Trades` タブを開く
2. CSV export 操作を行う
3. ファイル名は任意だが、運用上は `list_of_trades_*.csv` を推奨する

用途:

- トレード行から summary を再計算したい場合
- 日本語 UI / 英語 UI の両方に対応したい場合

### 3.5 英語 UI / 日本語 UI の扱い

- Performance Summary は現行実装では英語ヘッダー前提
- List of Trades は英語ヘッダー / 日本語ヘッダーに対応
- 日本語 UI の CSV を使う場合は、まず List of Trades を優先する

## 4. 北極星側で import する手順

### 4.1 Rule Lab から入る場合

1. `/strategy-lab` を開く
2. 自然言語ルールから strategy / version / Pine を生成する
3. backtest を自動作成した後、CSV ファイルを選択する
4. `CSV を取込` を押す
5. import 結果を画面上で確認する
6. 必要なら `バックテストレポートを開く` から `/backtests/:backtestId` へ移動する

### 4.2 Backtest Detail から確認する場合

1. `/backtests/:backtestId` を開く
2. `最新取込` ブロックで `parse_status / parse_error / parsed_summary` を確認する
3. `主要指標` ブロックで summary を確認する
4. `import 履歴` ブロックで過去の parsed / failed を確認する

## 5. 北極星が受け付ける CSV 形式

## 5.1 Performance Summary（英語ヘッダー）

必須ヘッダー:

- `Net Profit`
- `Total Closed Trades`
- `Percent Profitable`
- `Profit Factor`
- `Max Drawdown`
- `From`
- `To`

備考:

- 1 行目を header、2 行目を value として読む
- UTF-8 BOM を許容する

## 5.2 List of Trades（英語ヘッダー）

必須ヘッダー:

- `Trade #` または `Trade`
- `Type`
- `Date/Time` または `Date`
- `Net Profit` または `Profit`
- `Cumulative Profit`

## 5.3 List of Trades（日本語ヘッダー）

必須ヘッダー:

- `トレード番号`
- `タイプ`
- `日付` または `日付/時刻`
- `損益 JPY`
- `累積損益 JPY`

## 5.4 文字コード / BOM / 改行

- UTF-8 を前提にする
- UTF-8 BOM を許容する
- LF / CRLF を許容する

## 5.5 非対応として扱うもの

- Overview 系 CSV
- chart export CSV
- Broker / Paper Trading の取引履歴 CSV

## 6. 成功時の確認手順

### 6.1 取込直後

- Rule Lab:
  - `parse_status`
  - `parse_error`
  - `parsed_summary`
- Backtest Detail:
  - `最新取込`
  - `主要指標`
  - `import 履歴`

### 6.2 Backtest Detail で見る場所

確認順:

1. `最新取込`
   - `parse_status = parsed`
   - `parse_error = null`
2. `主要指標`
   - `totalTrades`
   - `winRate`
   - `profitFactor`
   - `maxDrawdown`
   - `netProfit`
   - `periodFrom`
   - `periodTo`
3. `import 履歴`
   - file 名
   - parsed / failed

### 6.3 AI 総評と比較へ進むタイミング

- parsed import が 1 件以上ある時点で AI 総評生成へ進める
- parsed import が 2 件以上ある時点で inline comparison / saved comparison へ進める

## 7. 失敗時の確認手順

### 7.1 Rule Lab の失敗表示

まず以下を確認する。

- `parse_status = failed`
- `parse_error`
- 補助文言

現行補助文言の観点:

- 空 CSV
- header + data row 不足
- required columns 不足
- 対応形式外 CSV

### 7.2 HTTP エラーの見方

- `400`
  - 入力内容 / CSV 形式 / 必須項目不足を疑う
- `413`
  - ファイルまたは `csv_text` が大きすぎる
- `415`
  - 送信形式や Content-Type が想定と違う

### 7.3 latest failed + parsed あり の見方

Backtest Detail では、最新 import が failed でも過去 parsed import が残っていれば補助文言を表示する。

運用上の見方:

- `latest_import` は直近の失敗を示す
- `import 履歴` は過去の成功 / 失敗を並べて見る
- 主要指標 / inline comparison / AI 総評では parsed 済みデータを確認できる

## 8. 再 import の扱い

- 同じ backtest に CSV を再 import すると、`imports` に履歴が追加される
- 上書きではなく履歴追加で扱う
- `latest_import` は常に最新の 1 件

推奨運用:

1. Performance Summary を 1 件 import
2. 必要なら List of Trades を同じ backtest に追加 import
3. 比較対象が増えたら inline comparison / saved comparison で確認

## 9. fixture / test で固定している内容

- Performance Summary 英語ヘッダー import
- List of Trades 英語ヘッダー import
- List of Trades 日本語ヘッダー import
- UTF-8 BOM 付き CSV import
- invalid CSV import
- latest failed + parsed ありの見方

## 10. 注意

- 実 CSV には個人情報や口座情報を含めない
- docs や PR に raw CSV 全文を貼らない
- TradingView の UI 表記は `Strategy Report / Strategy Tester` で揺れる可能性がある
- 北極星側の受入れ形式は現行 parser 実装を正とする
