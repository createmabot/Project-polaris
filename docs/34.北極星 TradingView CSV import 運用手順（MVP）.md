# 北極星 TradingView CSV import 運用手順（MVP）
更新日: 2026-05-05

## 1. 目的
本ドキュメントは、TradingView の Strategy Report / Strategy Tester から出力した CSV を、北極星の Rule Lab / Backtest 画面へ取り込む実運用手順を固定するための runbook です。

役割分担は次のとおりです。
- TradingView: Pine の貼り付け、チャート上での一次検証、CSV export
- 北極星: CSV import、取込履歴の保存、比較、AI総評、履歴管理

## 2. 前提
- Pine Script は TradingView 上で一次検証する
- 北極星では `POST /api/backtests/:backtestId/imports` に JSON body で `csv_text` を送って取り込む
- import 導線は `/strategy-lab` または `/backtests/:backtestId`
- backend の bodyLimit は 2 MB
- 現行で受け付ける CSV は次の 3 種類
  - Performance Summary 英語ヘッダー
  - List of Trades 英語ヘッダー
  - List of Trades 日本語ヘッダー

## 3. TradingView 側で CSV を出す手順
### 3.1 Pine Script を開く
1. TradingView で対象チャートを開く
2. Pine Script を貼り付ける、または既存 strategy を開く
3. strategy として実行する

### 3.2 Strategy Report / Strategy Tester を開く
- TradingView の UI では `Strategy Report` と `Strategy Tester` の表記揺れがある
- 北極星ではどちらの表記でも、バックテスト結果の CSV を出せる画面として扱う

### 3.3 Performance Summary を export する場合
1. `Performance Summary` タブを開く
2. CSV export 操作を行う
3. ファイル名は管理しやすいように `performance_summary_*.csv` とする

運用メモ:
- 北極星がそのまま受け付けるのは、次の英語ヘッダーがある CSV
- `Net Profit / Total Closed Trades / Percent Profitable / Profit Factor / Max Drawdown / From / To`

### 3.4 List of Trades を export する場合
1. `List of Trades` タブを開く
2. CSV export 操作を行う
3. ファイル名は管理しやすいように `list_of_trades_*.csv` とする

運用メモ:
- トレード一覧から summary を再計算したい場合に使う
- 日本語 UI / 英語 UI の両方に対応している

### 3.5 英語 UI / 日本語 UI の違い
- Performance Summary は現行実装では英語ヘッダーのみ対応
- List of Trades は英語ヘッダー / 日本語ヘッダーに対応
- 日本語 UI の CSV を使う場合は、まず List of Trades を優先する

## 4. 北極星側で import する手順
### 4.1 Rule Lab から入る場合
1. `/strategy-lab` を開く
2. 自然言語ルールから strategy / version / Pine を生成する
3. backtest を作成した後、CSV ファイルを選択する
4. `CSV を取り込む` を押す
5. import 結果を画面上で確認する
6. 必要なら `バックテストレポートを開く` から `/backtests/:backtestId` へ移動する

### 4.2 Backtest Detail から確認する場合
1. `/backtests/:backtestId` を開く
2. `最新取込` ブロックで `parse_status / parse_error / parsed_summary` を確認する
3. `主要指標` ブロックで summary を確認する
4. `import 履歴` ブロックで過去の parsed / failed を確認する

## 5. 北極星が受け付ける CSV 形式
### 5.1 Performance Summary（英語ヘッダー）
必須ヘッダー:
- `Net Profit`
- `Total Closed Trades`
- `Percent Profitable`
- `Profit Factor`
- `Max Drawdown`
- `From`
- `To`

補足:
- 1 行目が header、2 行目が value であることを前提に読む
- UTF-8 BOM 付きでも取り込み可能

### 5.2 List of Trades（英語ヘッダー）
必須ヘッダー:
- `Trade #` または `Trade`
- `Type`
- `Date/Time` または `Date`
- `Net Profit` または `Profit`
- `Cumulative Profit`

### 5.3 List of Trades（日本語ヘッダー）
必須ヘッダー:
- `トレード番号`
- `タイプ`
- `日時`
- `純損益 JPY`
- `累積損益 JPY`

### 5.4 文字コード / BOM / 改行
- UTF-8 を前提にする
- UTF-8 BOM を受け付ける
- LF / CRLF を受け付ける

### 5.5 受け付け対象外として扱うもの
- Overview 系 CSV
- chart export CSV
- Broker / Paper Trading の別形式 CSV

## 6. 成功時の確認手順
### 6.1 確認場所
- Rule Lab
  - `parse_status`
  - `parse_error`
  - `parsed_summary`
- Backtest Detail
  - `最新取込`
  - `主要指標`
  - `import 履歴`

### 6.2 Backtest Detail で見る内容
1. `最新取込`
   - `parse_status = parsed`
   - `parse_error = null`
2. `主要指標`
   - `totalTrades`
   - `winRate`
   - `profitFactor`
   - `maxDrawdown`
   - `netProfit`
   - `periodFrom`
   - `periodTo`
3. `import 履歴`
   - file 名
   - parsed / failed

### 6.3 AI 総評と比較へ進むタイミング
- parsed import が 1 件以上あれば AI 総評生成へ進める
- parsed import が 2 件以上あれば inline comparison / saved comparison へ進める

## 7. 失敗時の確認手順
### 7.1 Rule Lab の失敗表示
まず次を確認する。
- `parse_status = failed`
- `parse_error`
- 補助説明文

代表的な補助説明文の観点:
- 空 CSV
- header + data row 不足
- required columns 不足
- 対応形式外の CSV

### 7.2 HTTP エラーの意味
- `400`
  - 入力形式 / CSV 形式 / 必須項目不足の可能性
- `413`
  - ファイルまたは `csv_text` が大きすぎる
- `415`
  - 送信形式や Content-Type が想定と違う

### 7.3 latest failed + parsed あり の見方
Backtest Detail では、最新 import が failed でも過去 parsed import が残っていれば補助説明文を表示する。

運用上の見方:
- `latest_import` は直近の失敗を示す
- `import 履歴` は過去の成功 / 失敗を並べて見る
- 主要指標 / inline comparison / AI 総評では parsed 済みデータを確認できる

## 8. 再 import の扱い
- 同じ backtest に CSV を再 import すると、`imports` に履歴が追加される
- 上書きではなく履歴追加で扱う
- `latest_import` は常に最新の 1 件

運用確認:
1. Performance Summary を 1 件 import
2. 必要なら List of Trades を同じ backtest に追加 import
3. 比較対象が揃ったら inline comparison / saved comparison で確認

## 9. fixture / test で固定している観点
- Performance Summary 英語ヘッダー import
- List of Trades 英語ヘッダー import
- List of Trades 日本語ヘッダー import
- UTF-8 BOM 付き CSV import
- invalid CSV import
- latest failed + parsed あり の表示

## 10. 注意
- 実 CSV には個人情報や口座情報を含めない
- docs や PR に raw CSV 全文を載せない
- TradingView の UI 表記は `Strategy Report / Strategy Tester` で揺れる可能性がある
- 現行 parser の受け付け名を正とする
- `日付` / `日付/時刻` / `損益 JPY` が実TradingView日本語CSVで出ることを確認できた場合のみ、alias追加を別タスクで判断する
- 北極星側の受け付け形式を変える場合は、先に parser 実装を正とする
