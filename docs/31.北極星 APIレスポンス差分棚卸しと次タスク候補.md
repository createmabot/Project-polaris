# 北極星 APIレスポンスキー 実装差分棚卸しと次タスク候補

更新日: 2026-04-26

## 1. 目的

「docs/16.北極星 APIごとの JSON サンプル集（MVP）.md」および関連ドキュメントに記載されているAPIレスポンスキーと、現行実装（backend / frontend types）の間に生じている差分を棚卸しし、分類する。

最重要前提として「docs配下を正本とする」が、実装を不用意にdocsへ合わせて破壊的変更（フロントエンドの大規模改修など）を引き起こさないよう、まずは差分を分類し、安全にdocsを更新できるものと、次タスク（バックエンド・フロントエンド改修）として回すものに整理する。

## 2. 差分分類と対応方針

差分は以下の基準で分類した。

*   **A. Docs out of date (ドキュメント更新漏れ):** 実装やフロントエンドの要件定義が先行し、docsに反映されていないもの。新フィールドの追加や、より適切な構造化（ネスト等）が行われているケース。破壊的変更はないため、**docsを実装に合わせて更新する**。
*   **C. Implementation Drift (実装の逸脱):** docsの設計から実装が乖離しているが、すでにフロントエンドがその実装に依存して安定稼働しているケース。不用意に実装をdocsに戻すと破壊的変更となるため、今回は実装は修正せず、**次タスク候補として記録するか、docs側を現状に合わせて正とする**。
*   **D. Undocumented (未定義):** docsに記載がないが、実装上必要なAPIとして存在しているもの。**docsに追記する**。

## 3. 棚卸し結果（ドメイン別）

### 3.1 Home
- **エンドポイント:** `GET /api/home`
- **差分:** なし。完全に一致。
- **分類:** Consistent
- **対応:** 対応不要。

### 3.2 SymbolDetail
- **エンドポイント:** `GET /api/symbols/:id`
- **差分1:** docsの `latest_note_summary` (id, title) に対し、実装は `latest_active_note` (note全体のフィールド) を返す。
- **分類:** C. Implementation Drift
- **対応:** 実装を戻すとフロントエンドが壊れるため、今回は実装を正とし、docsを `latest_active_note` に更新する。
- **差分2:** docsの `latest_ai_thesis_summary` は `summary_id` を持つが、実装は `id`, `body_markdown`, `overall_view`, `structured_json` 等詳細を持つ。
- **分類:** A. Docs out of date
- **対応:** docsを実装に合わせて更新。
- **差分3:** 実装には `chart`, `recent_alerts`, `related_references`, `latest_processing_status` が追加されている。
- **分類:** A. Docs out of date
- **対応:** docsに追記。

- **エンドポイント:** `GET /api/symbols/:id/ai-summary`
- **差分:** docsは flat なキーを `data` 直下に置くが、実装は `data: { symbol_id, scope, summary: { ... } }` のネスト構造。
- **分類:** A. Docs out of date
- **対応:** docsを実装のネスト構造に合わせて更新。

- **エンドポイント:** `POST /api/symbols/:id/ai-summary/generate`
- **差分:** docsは `job_id`, `status` のみ。実装は `symbol_id`, `scope`, `job_id`, `status`, `summary` を返す。
- **分類:** A. Docs out of date
- **対応:** docsを更新。

### 3.3 Comparison
- **エンドポイント:** `POST /api/comparisons`
- **差分:** docsは `comparison_id`, `symbols`。実装は `comparison_session`, `comparison_symbols`。
- **分類:** A. Docs out of date
- **対応:** docsを実装のキー名に更新。

- **エンドポイント:** `GET /api/comparisons/:id`
- **差分:** 概ね一致するが、実装は `latest_result` の中に `ai_summary_id` を配置し、ヘッダにメタデータ (`symbol_count`, `updated_at`) が追加されている。
- **分類:** A. Docs out of date
- **対応:** docsを更新。

### 3.4 Alerts
- **エンドポイント:** `GET /api/alerts/:id/summary`
- **差分:** docs/16 に未記載。実装は `alert_id`, `summary`, `latest_job` を返す。
- **分類:** D. Undocumented
- **対応:** docs/16 に追記。

### 3.5 Rule Lab / Strategy / Pine
- **エンドポイント:** `GET /api/strategies/:strategyId/versions/:versionId/pine`
- **差分:** 実装は docs に記載の項目に加え、`script_body` と `generated_at` を返却する。
- **分類:** A. Docs out of date
- **対応:** docsに追記。`generated_script` と `script_body` は互換性のため両方維持（フロントエンドが利用する可能性があるため）。

### 3.6 Backtest
- **エンドポイント:** `GET /api/backtests/:id` および `GET /api/backtests/:id/imports`
- **差分:** docs/16 に未記載。実装は `backtest`, `used_strategy`, `latest_import`, `imports`, `ai_review` などを返す。
- **分類:** D. Undocumented
- **対応:** docs/16 に追記。

## 4. 次タスク候補（破壊的変更を伴う可能性があり見送ったもの）

今回の棚卸しにおいて、**「本来はdocsの設計が正しく、実装をそれに合わせるべきだが、現行のフロントエンドに破壊的変更をもたらすため見送った」という深刻なケース（Category B相当）は発見されなかった**。

ほとんどの差分は「実装の進化に伴い、フィールドが追加されたり、より適切なネスト構造に変更されたが、docsの更新が漏れていた」という（Category A / C / D）ケースであった。

したがって、今回のスプリントでは、**安全な対応として「docs/16.md」側を現行の実装（真のSource of Truth）に合わせて更新する**作業を実施する。

※もし将来的にAPIレスポンスの完全なスネークケース統一や、冗長キー（`generated_script` と `script_body` の重複など）のクリーンアップを行う場合は、BFF層の導入やAPIのv2化など、破壊的変更を許容するフェーズで実施することを推奨する。
