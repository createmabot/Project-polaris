# 北極星 TradingView 設定手順書（MVP）

更新日: 2026-03-15

## 1. このドキュメントの目的

本ドキュメントは、株価評価AIツール「北極星」と TradingView を連携させるための **実運用向け設定手順** をまとめたものである。

北極星のMVPでは、TradingView を以下の用途で利用する。

- チャート表示
    
- 価格 / テクニカル / watchlist アラート
    
- strategy を使った一次検証
    
- order fill alert を使った運用補助
    

この手順書では特に、北極星側 webhook と接続するための TradingView 設定に焦点を当てる。

---

## 2. この手順書の対象

MVPで対象とする設定は以下。

1. webhook 利用前提の準備
    
2. 通常アラートの設定
    
3. watchlist alert の設定
    
4. strategy / order fill alert の設定
    
5. 北極星向け alert message テンプレート設定
    
6. 動作確認方法
    
7. よくある失敗と対処
    

---

## 3. 事前準備

## 3.1 北極星側で準備しておくもの

TradingView 側設定の前に、北極星側で以下を準備しておく。

- webhook endpoint URL
    
- webhook token
    
- 必要なら shared_secret
    
- 推奨 JSON message テンプレート
    
- symbol マスタ（少なくとも監視銘柄分）
    

### 例

- webhook URL: `https://your-domain.example/api/integrations/tradingview/webhook?token=tv_xxxxx`
    
- shared_secret: `tv_shared_secret_xxx`
    

---

## 3.2 TradingView 側の事前条件

- TradingView アカウントにログイン済み
    
- 対象プランで alert が利用可能
    
- webhook alerts を使う場合は 2FA を有効化しておく
    
- Supercharts にアクセスできる
    

---

## 3.3 北極星側の前提設定

以下の方針を前提にする。

- webhook は JSON payload を標準とする
    
- alert message は北極星テンプレートに従う
    
- symbol 解決のため `tradingview_symbol` / `symbol` / `market_code` を含める
    
- 価格は `trigger_price` に統一する
    
- alert 種別は `alert_type` に統一する

---

## 4. 基本設定フロー

北極星と TradingView を接続する基本フローは以下。

1. 監視したい銘柄または strategy を決める
    
2. TradingView で alert を作成する
    
3. webhook URL を設定する
    
4. 北極星用 JSON message を設定する
    
5. 保存する
    
6. テスト発火または条件待ちで受信確認する
    

---

## 5. webhook alerts を使う前の確認事項

## 5.1 webhook URL の確認

TradingView に設定する URL は、北極星側で公開されている HTTPS endpoint である必要がある。

MVPでは以下を守る。

- HTTPS を使う
    
- query parameter に token を含める
    
- 必要なら payload 内に shared_secret を含める
    

### 例

`https://your-domain.example/api/integrations/tradingview/webhook?token=tv_xxxxx`

---

## 5.2 endpoint 側の制約に合わせる

北極星の webhook endpoint は、次を満たす前提で運用する。

- 即時に 200 応答できること
    
- payload を受け取った後の重い処理は非同期化されていること
    
- JSON / text/plain の両方を受けられること
    

MVPでは JSON を標準運用にする。

Webhook 運用上の前提として以下を守る。

- 送信先 URL は HTTPS を使う
    
- 利用可能ポートは 80 / 443 のみとする
    
- endpoint は短時間で応答し、重い処理は非同期へ逃がす
    
- alert message は JSON を推奨する
    
- 機密情報を payload 本文に含めない

---

## 6. 通常アラート（価格 / テクニカル）の設定手順

## 6.1 用途

- 価格到達通知
    
- 移動平均線上抜け通知
    
- RSI閾値通知
    
- 出来高急増通知
    

---

## 6.2 手順

### 手順1

TradingView の Supercharts を開く。

### 手順2

対象銘柄を表示する。

### 手順3

「Create Alert」またはアラート作成 UI を開く。

### 手順4

Condition を設定する。

例:

- Price crossing up
    
- RSI > 70
    
- close crossing SMA25
    

### 手順5

Webhook URL を有効にし、北極星の endpoint URL を入力する。

### 手順6

Message 欄に、北極星向け JSON テンプレートを貼り付ける。

### 手順7

保存する。

---

## 6.3 推奨テンプレート（通常アラート）

```json
{
  "provider": "tradingview",
  "alert_type": "technical",
  "alert_name": "MA25 breakout",
  "tradingview_symbol": "{{exchange}}:{{ticker}}",
  "symbol": "{{ticker}}",
  "market_code": "{{exchange}}",
  "timeframe": "{{interval}}",
  "triggered_at": "{{time}}",
  "trigger_price": {{close}},
  "currency": "{{syminfo.currency}}",
  "message": "{{ticker}} triggered technical alert",
  "condition_summary": "close crossed above MA25",
  "metadata": {
    "open": {{open}},
    "high": {{high}},
    "low": {{low}},
    "volume": {{volume}}
  }
}
```

---

## 6.4 運用メモ

- `alert_name` は意味が分かる名前にする
    
- `condition_summary` は固定文字列でもよい
    
- `{{interval}}` は補助値として扱う
    
- 値動きの根拠説明は北極星側で行うため、TradingView message は事実中心でよい
    

---

## 7. watchlist alert の設定手順

## 7.1 用途

- 監視銘柄群に同じ条件を適用したいとき
    
- 毎日5分の状況把握を効率化したいとき
    

例:

- 監視銘柄のうち前日比 +5% 到達
    
- 監視銘柄のうち出来高急増
    
- 監視銘柄のうちRSI閾値突破
    

---

## 7.2 手順

### 手順1

TradingView の watchlist を作成または選択する。

### 手順2

watchlist のメニューから「Add alert on the list…」相当の導線を選ぶ。

### 手順3

対象 watchlist を確認する。

### 手順4

適用したい条件を設定する。

### 手順5

Webhook URL を設定する。

### 手順6

watchlist 用 JSON message テンプレートを設定する。

### 手順7

保存する。

---

## 7.3 推奨テンプレート（watchlist alert）

```json
{
  "provider": "tradingview",
  "alert_type": "watchlist",
  "alert_name": "Watchlist momentum signal",
  "watchlist_name": "JP Core Watchlist",
  "tradingview_symbol": "{{exchange}}:{{ticker}}",
  "symbol": "{{ticker}}",
  "market_code": "{{exchange}}",
  "timeframe": "{{interval}}",
  "triggered_at": "{{time}}",
  "trigger_price": {{close}},
  "currency": "{{syminfo.currency}}",
  "message": "{{ticker}} matched watchlist condition",
  "condition_summary": "watchlist momentum condition matched",
  "metadata": {
    "volume": {{volume}}
  }
}
```

---

## 7.4 watchlist alert 運用ルール

- watchlist 名を固定し、北極星側の watchlist 名称とできるだけ合わせる
    
- 1つの watchlist alert で多くを抱えすぎない
    
- 条件は意味の分かる粒度にする
    
- 初期は「価格急変」「出来高急増」「基本テクニカル」の 2〜3 個に絞る
    

---

## 8. strategy / order fill alert の設定手順

## 8.1 用途

- Pine strategy のフォワード監視
    
- strategy シグナルを webhook で北極星へ送る
    
- order fill 単位の通知を行う
    

---

## 8.2 前提

strategy alert を使う前に、Pine strategy が次を満たしていることを確認する。

- strategy としてコンパイルされる
    
- entry / exit が定義されている
    
- 必要なら `alert_message` が strategy.entry / strategy.exit / strategy.close などに設定されている
    

---

## 8.3 手順

### 手順1

TradingView に生成済み strategy を表示する。

### 手順2

Create Alert を開く。

### 手順3

Condition で strategy を選ぶ。

### 手順4

「Order fills only」または「Order fills and alert() function calls」のいずれかを選ぶ。

MVPではまず `Order fills only` を推奨する。

### 手順5

Webhook URL を設定する。

### 手順6

Message に strategy 用テンプレートを設定する。

### 手順7

保存する。

---

## 8.4 推奨テンプレート（strategy / order fill）

```json
{
  "provider": "tradingview",
  "alert_type": "strategy",
  "alert_name": "Strategy order fill",
  "strategy_name": "MA25 Swing Strategy",
  "tradingview_symbol": "{{exchange}}:{{ticker}}",
  "symbol": "{{ticker}}",
  "market_code": "{{exchange}}",
  "timeframe": "{{interval}}",
  "triggered_at": "{{timenow}}",
  "trigger_price": {{strategy.order.price}},
  "currency": "{{syminfo.currency}}",
  "message": "{{strategy.order.alert_message}}",
  "condition_summary": "strategy order fill",
  "metadata": {
    "order_action": "{{strategy.order.action}}",
    "order_id": "{{strategy.order.id}}",
    "contracts": "{{strategy.order.contracts}}",
    "position_size": "{{strategy.position_size}}",
    "market_position": "{{strategy.market_position}}"
  }
}
```

---

## 8.5 Pine 側の推奨書き方

北極星で生成する Pine では、少なくとも entry / exit 側に `alert_message` を埋めておく。

### 例

```pine
if entryCondition and strategy.position_size == 0
    strategy.entry("Long", strategy.long, alert_message = "entry_long")

if exitCondition and strategy.position_size > 0
    strategy.close("Long", alert_message = "exit_long")
```

---

## 8.6 strategy alert 運用ルール

- Message 欄に `{{strategy.order.alert_message}}` を必ず含める
    
- entry と exit の alert_message は意味が分かる値にする
    
- 実運用前は少数銘柄・短期間でテストする
    

---

## 9. アラート命名規則

MVPでは alert_name の命名規則を持つ。

## 9.1 推奨ルール

- 内容が分かる短い名称にする
    
- `test` や `alert1` は避ける
    
- strategy と通常アラートを区別しやすくする
    

## 9.2 例

- `MA25 breakout`
    
- `RSI over 70`
    
- `Volume spike`
    
- `Watchlist momentum signal`
    
- `Strategy order fill`

## 10. webhook 動作確認手順

## 10.1 受信確認の流れ

1. TradingView で alert を保存する
    
2. 条件を満たす、またはテスト環境で発火を待つ
    
3. 北極星側の alert_events を確認する
    
4. event の symbol 解決結果を確認する
    
5. external references 収集ジョブが動いたか確認する
    
6. AI要約が生成されたか確認する
    

---

## 10.2 最低限確認すべき項目

- HTTP 受信成功
    
- alert_event 作成
    
- symbol 解決成功
    
- processing_status の遷移
    
- raw payload 保存
    
- 期待する alert_kind で受理されている
    

---

## 11. 典型的な初期セットアップ手順

MVPで最初に作るとよい alert は次の順番。

### 11.1 Step 1

1銘柄だけで通常アラートを設定する。

例:

- 7203 の MA25 breakout
    

目的:

- webhook 疎通確認
    
- symbol 解決確認
    

### 11.2 Step 2

1つの watchlist alert を設定する。

例:

- 日本株監視リストに対する volume spike
    

目的:

- 複数銘柄監視の動作確認
    

### 11.3 Step 3

1つの strategy alert を設定する。

例:

- MA25 strategy order fill
    

目的:

- natural language → Pine → TradingView → webhook の流れ確認

## 12. よくある失敗と対処

## 12.1 Message が JSON として壊れている

### 症状

- 北極星側で text/plain として受信される
    
- JSON parse 失敗
    

### 対処

- ダブルクォートを確認する
    
- 数値部分に余計な引用符がないか確認する
    
- 最初はテンプレートをコピペで使う
    

---

## 12.2 symbol 解決に失敗する

### 症状

- `processing_status=unresolved_symbol`
    

### 対処

- `tradingview_symbol` の形式を確認する
    
- `exchange` と `symbol` が期待通りか確認する
    
- 北極星側 symbol マスタに登録されているか確認する
    

---

## 12.3 strategy alert で message が空になる

### 症状

- `message` が空
    
- order fill の意味が取れない
    

### 対処

- Pine 側で `alert_message` を設定しているか確認する
    
- Create Alert の Message に `{{strategy.order.alert_message}}` を入れているか確認する
    

---

## 12.4 watchlist alert が想定より発火しない

### 症状

- 監視銘柄の一部でしか発火しないように見える
    

### 対処

- 条件が厳しすぎないか確認する
    
- watchlist に対象銘柄が含まれているか確認する
    
- 通常アラートで同条件が出るか比較する
    

---

## 12.5 webhook は届くが要約が出ない

### 症状

- alert_event はあるが AI要約が生成されない
    

### 対処

- external references 収集ジョブ状態を確認する
    
- `insufficient_context` で止まっていないか確認する
    
- ai_job の failed を確認する
    

---

## 13. 推奨運用ルール

## 13.1 最初は alert 種別を増やしすぎない

MVP初期は以下で十分。

- 通常アラート 1〜2個
    
- watchlist alert 1個
    
- strategy alert 1個
    

## 13.2 Message テンプレートは使い回す

alert ごとに payload 構造を変えない。

## 13.3 命名を揃える

- alert_name
    
- watchlist_name
    
- strategy_name  
    を北極星側の表示と対応づけやすくする。
    

## 13.4 strategy alert は実運用前に sandbox 的に試す

特に order fill 連携は entry / exit の意味が正しく届くか先に確認する。

---

## 14. セキュリティ / 運用注意

- webhook body に認証情報や秘密情報を書き込まない
    
- token は URL で使うが、必要ならローテーションできるようにする
    
- shared_secret を追加する場合は payload に平文で入れる前提を理解する
    
- 北極星側で raw payload は保存するが、表示時はマスク方針を持つ
    

---

## 15. 北極星MVPでの推奨セットアップ順

1. 通常アラート 1件で疎通確認
    
2. watchlist alert 1件で複数監視確認
    
3. strategy alert 1件で rule 検証フロー確認
    
4. その後、日常運用に必要な alert を追加

## 17. 結論

北極星MVPにおける TradingView 設定では、  
**「通常アラート」「watchlist alert」「strategy / order fill alert」** の3系統を標準運用とし、  
すべてを北極星向け JSON テンプレートで webhook 送信する方針を採用する。

そのため、MVPでは以下を重視する。

- webhook URL と token を正しく設定する
    
- JSON message を壊さずに設定する
    
- symbol 解決に必要なキーを必ず含める
    
- strategy alert では `{{strategy.order.alert_message}}` を必ず使う
    
- 少数の alert から段階的に運用を始める
    

この手順に沿うことで、TradingView と北極星の連携を安定して立ち上げることができる。
