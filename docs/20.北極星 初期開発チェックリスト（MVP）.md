# 北極星 初期開発チェックリスト（MVP）

更新日: 2026-03-18

## 1. このドキュメントの目的

本ドキュメントは、株価評価AIツール「北極星」のMVP開発に着手する前後で確認すべき事項を、**実務用チェックリスト** として整理するものである。

これまで、企画方針、画面設計、データモデル、API、webhook、AI要約、自然言語→Pine、CSVパース、比較、実装チケット、スプリント計画まで整理してきた。

一方で、実装開始時には以下のような初歩的だが重大な抜け漏れが起きやすい。

- 開発環境が統一されていない
    
- symbol seed が足りず webhook が正しく解決できない
    
- TradingView テスト条件が未整備
    
- ローカルLLMと fallback API の役割分担が曖昧
    
- fixtures がないためテストが不安定
    
- どこまでできたら Sprint 1 着手可能かが曖昧
    

この資料では、開発初期に確認すべきことを段階別にチェック可能な形でまとめる。

# 20. 結論

北極星MVPの初期開発では、仕様が揃っていても、  
**環境・seed・TradingView・AIルーティング・fixtures・ログ** のどれかが欠けると実装が詰まりやすい。

そのため、開発着手前には以下を重点確認する。

- webhook が試せる状態か
- symbol 解決に必要な seed があるか
- ローカルLLMと fallback API の役割分担が決まっているか
- AI と非同期ジョブの疎通があるか
- parser / webhook の fixtures があるか
- Sprint 1 の完了条件が明確か

このチェックリストを使うことで、北極星MVPの立ち上がりを安定させ、初期の手戻りを減らすことを目指す。

## Prisma migration drift check（追加）

- [ ] `cd backend && npx prisma validate` が通る
- [ ] `cd backend && npx prisma migrate status` が `Database schema is up to date!` を返す
- [ ] `cd backend && npx prisma migrate dev` が `Already in sync` で完了する
- [ ] `cd backend && npx prisma migrate deploy` が `No pending migrations` で完了する
- [ ] checksum 不一致（`migration ... was modified after it was applied`）時は、repo 正本を優先してローカル開発DBを `migrate reset --force --skip-seed` で再生成する
- [ ] Prisma 設定の正本が backend/prisma.config.ts であることを確認する（package.json#prisma は使わない）
