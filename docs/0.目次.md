## 7. docs 更新ルール

- 新しい機能判断は、該当する正本 docs に追記する。
- 古い docs を直接正本化し直すより、`docs/53` または個別正本 docs へ現在地を追記する。
- 大量削除 / rename は行わず、必要な場合は先に `docs/57` へ後続判断として記録する。
- PR本文は ASCII English only、docs本文は UTF-8 日本語で保存する。
- local path、secret、token、shared_secret、API key の実値を docs / PR / log に残さない。
