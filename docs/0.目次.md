# 北極星 docs 目次・読む順番

更新日: 2026-05-17

## 1. この目次の役割

`docs/` は北極星プロジェクトの仕様・運用判断の正本です。
現行 docs は次の3分類で読みます。

1. `docs/仕様書/`: 現在のシステム仕様・画面仕様・データ仕様・API仕様・AI仕様・UI component仕様。
2. `docs/運用ドキュメント/`: 開発・起動・テスト・運用・確認・トラブルシュート手順。
3. `docs/作業進捗管理/`: ロードマップ、フェーズ完了整理、残課題、作業進捗、decision log。

MVP初期からの平置き docs は原則として履歴資料として残します。削除済み・保留中の番号付き docs は `docs/作業進捗管理/06_番号付きdocs削除整理.md` の記録に沿って扱います。現行仕様の入口は `docs/仕様書/`、分類詳細は `docs/57` を優先してください。

- docs体系と分類の正本: `docs/57.北極星 docs正本整理・読む順番（現行）.md`
- 開発者向け入口: `README.md`
- 動作確認・受け入れ確認: `docs/walkthrough.md`

## 8. docs 更新ルール

- 現行仕様は `docs/仕様書/` に追加・更新する。
- 開発・確認・運用手順は `docs/運用ドキュメント/` に追加・更新する。
- roadmap / backlog / decision log / completion summary は `docs/作業進捗管理/` または `docs/53` に追加・更新する。
- 平置き旧docsの削除 / rename は `docs/作業進捗管理/06_番号付きdocs削除整理.md` の削除済み一覧、保留理由、後続判断に沿って別PRで判断する。
- PR本文は ASCII English only、docs本文は UTF-8 日本語で保存する。
- local path、secret、token、shared_secret、API key の実値を docs / PR / log に残さない。
