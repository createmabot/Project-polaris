# 北極星 Home card 表示件数・並び順・snapshot依存仕様（MVP後P1）

## 1. 目的

- Home の `watchlist_symbols` / `positions` card について、何が・何件・どの順番で・snapshot が無い場合にどう表示されるべきかを固定する。
- `watchlist / positions API CRUD 成立` と `Home card 上での即時視認` を分けて扱い、MVP後P1 の受入条件を明確にする。
- Home は詳細管理画面ではなく、状況把握の入口であることを明文化する。

## 2. 現行実装の確認対象

- backend
  - `backend/src/routes/home.ts`
  - `backend/src/home/positions-read-model.ts`
  - `backend/src/routes/watchlist.ts`
  - `backend/src/routes/positions.ts`
- frontend
  - `frontend/src/pages/Home.tsx`
- tests
  - `backend/test/home.e2e.test.ts`
  - `frontend/src/pages/Home.test.tsx`

## 3. Home card の基本方針

- Home は詳細管理画面ではなく、日次の状況把握と詳細導線の入口を担う。
- `watchlist` と `positions` の正本は、管理画面と DB / API 側にある。
- Home card に表示されないことと、CRUD が失敗していることは同一ではない。
- `Home反映` は次の 3 段階で判定する。
  1. API 反映
  2. Card 視認
  3. Detail 導線
- snapshot 不足は、Home CRUD 反映失敗ではなく、価格系表示の不足として扱う。

## 4. watchlist_symbols の現行挙動と期待仕様

### 4.1 対象範囲

- `GET /api/home` は default watchlist 1 件を対象にする。
- 対象 watchlist は `watchlist.sortOrder asc`, 同順位では `createdAt asc` で先頭のもの。

### 4.2 表示件数

- backend / frontend ともに現時点では件数上限を設けていない。
- `GET /api/home` の `watchlist_symbols` は、対象 watchlist の item を全件返す。
- Home card も backend から返った配列をそのまま表示する。

### 4.3 並び順

- DB 取得は `addedAt asc`。
- 返却前に次の優先順で並び替える。
  1. `priority asc`
  2. `priority = null` は最後
  3. 同 priority では `addedAt asc`
- `priority` は表示順の主キーであり、`addedAt` は同 priority の tie-breaker。

### 4.4 snapshot が無い場合の扱い

- snapshot が無いことを理由に row を落とさない。
- row 自体は `watchlist_symbols` に残す。
- `latest_price` / `change_rate` は `null` を返す。
- Home UI は `null` を `-` 表示にする。

### 4.5 latest alert status

- `latest_alert_status` は、その symbol に紐づく最新 alert の `processingStatus` を使う。
- 最新 alert 判定は `triggeredAt desc` 基準。
- alert が無い場合は `null`。

### 4.6 導線

- Home card から SymbolDetail に到達できることを必須とする。
- watchlist の詳細管理は `/watchlist` を正本導線とする。

## 5. positions の現行挙動と期待仕様

### 5.1 対象範囲

- `GET /api/home` は positions read model に存在する row を対象にする。
- positions は transactions 正本であり、read model は `backend/src/home/positions-read-model.ts` で再構築される。

### 5.2 再構築タイミング

- `/api/home` 実行時に `rebuildPositionsReadModel()` を呼ぶ。
- `/api/positions` の create / update / delete でも user 単位で再構築される。

### 5.3 表示件数

- backend / frontend ともに件数上限は設けていない。
- `GET /api/home` の `positions` は read model の全件を返す。
- Home card も返却配列をそのまま表示する。

### 5.4 並び順

- 現行の `GET /api/home` は `position.createdAt asc` で返す。
- つまり Home の positions card は、read model 上の作成順に近い並びをとる。
- 損益順や評価額順での並び替えは現行仕様に含めない。

### 5.5 snapshot が無い場合の扱い

- snapshot が無いことを理由に row を落とさない。
- `latest_price` は `null` を返す。
- `unrealized_pnl` は `latest_price` が計算できないため `null` を返す。
- Home UI は `latest_price` / `unrealized_pnl` を `-` 表示にする。

### 5.6 導線

- Home card から SymbolDetail に到達できることを必須とする。
- positions の詳細管理は `/positions` を正本導線とする。

## 6. 「Home反映」の受入条件

### 6.1 API反映

- watchlist / positions CRUD 後に `GET /api/home` の `watchlist_symbols` / `positions` に対象 row が返ること。
- snapshot が無い場合でも row が返ること。

### 6.2 Card視認

- Home card は backend 返却配列をそのまま表示する。
- したがって Card 視認可否は、次の要因の影響を受ける。
  - watchlist の `priority`
  - watchlist の `addedAt`
  - positions の `createdAt`
  - 既存 seed / 実データ混在
  - snapshot の有無による価格表示不足
- Card に見えないことだけで CRUD 失敗と判定しない。

### 6.3 Detail導線

- Home card 上に見えている場合は、そこから SymbolDetail に到達できること。
- もし Home card 上で即時視認できなくても、`/watchlist` または `/positions` から SymbolDetail に到達できれば、導線成立と判定する。

## 7. snapshot依存の扱い

- snapshot が無いことは row 非表示の理由にしない。
- snapshot が無い場合は次のように扱う。
  - `latest_price: null`
  - `change_rate: null`
  - `unrealized_pnl: null`
  - frontend 表示は `-`
- snapshot 不足と Home CRUD 反映失敗は分けて判断する。
- stale / `market_status = unknown` の全体整理は別 P2 タスクとする。

## 8. 現時点の運用解釈

- temp symbol を追加した直後に Home card 上で即時視認できない場合でも、次の順で確認する。
  1. 管理画面に row があるか
  2. `GET /api/home` に row が返っているか
  3. SymbolDetail に到達できるか
  4. snapshot が無いため価格だけ `-` になっていないか
- この順で成立していれば、まずは `Home反映 OK / Card視認は要確認` と扱う。

## 9. テスト固定方針

- backend
  - watchlist item を `priority asc / null last / addedAt asc` で返すこと
  - snapshot が無い watchlist row も返り、価格系が `null` になること
  - snapshot が無い positions row も返り、`latest_price` / `unrealized_pnl` が `null` になること
- frontend
  - `latest_price: null` を `-` 表示できること
  - `unrealized_pnl: null` を `-` 表示できること
  - SymbolDetail link と管理画面 link が維持されること

## 10. 今回やらないこと

- Home card に新たな件数制限を入れること
- 並び順を損益順や market cap 順に変更すること
- snapshot なし row を非表示にすること
- stale / `market_status unknown` の全体整理
- watchlist / positions の複数ポートフォリオ対応
