# 北極星 企画方針まとめ

更新日: 2026-03-15

## 1. プロジェクト概要

**名称**: 北極星  
**目的**: AIを活用して、個人投資における「状況把握」「銘柄研究」「ルール検証」を効率化する株価分析ツールを構築する。

本ツールは商材化を前提とせず、**個人利用**を目的とする。そのため、金融商品取引法や販売時のコンプライアンス対応は現時点では検討対象外とする。

---

## 2. 北極星のコンセプト

北極星が実現すべき価値は、以下の3本柱とする。

### 2.1 毎日5分で状況把握

指数・為替・セクター・保有銘柄・監視銘柄の変化要因を、AIが短時間で把握可能な形に要約する。

### 2.2 銘柄研究の高速化

決算・適時開示・ニュースの要点抽出、論点整理、比較表生成を自動化し、個別銘柄研究にかかる時間を削減する。

### 2.3 ルール運用の検証

売買ルールを自然言語から定義し、バックテストを行い、再現性を担保しながらレポート化する。

---

## 3. 基本方針

### 3.1 北極星の役割

北極星は単なるチャート閲覧ツールではなく、**情報を解釈し、整理し、蓄積するための意思決定支援ツール**として位置づける。

### 3.2 独自価値の置き場所

TradingViewを利用することで、可視化や監視などの汎用機能は外部に寄せる。一方で、北極星独自の価値は以下に置く。

- AIによる要約と論点整理
    
- 銘柄比較の自動化
    
- 投資仮説ノートの蓄積
    
- 自然言語からのルール定義
    
- バックテスト結果の保存・比較・再利用
    

### 3.3 設計原則

- **見る・監視する・一次検証する** は TradingView に寄せる
    
- **意味づける・残す・比較する** は 北極星 側で実装する
    
- TradingViewを便利な実行基盤として使うが、北極星の知見資産は北極星側に保持する

## 4. TradingView活用方針

## 4.1 採用方針

開発方針として、TradingViewとの連携は **B案（実用本命）** を採用する。

### B案の概要

TradingView の Widget を北極星に埋め込み、以下を分担する。

**TradingView 側**

- チャート表示
    
- スクリーニング
    
- 会社情報表示
    
- ファンダメンタル表示
    
- テクニカルサマリ表示
    
- ニュース表示
    
- アラート発火
    
- Pine Script による一次検証
    

**北極星 側**

- AI要約
    
- 論点整理
    
- 比較表生成
    
- 仮説ノート管理
    
- 自然言語→Pine変換
    
- バックテスト履歴管理
    
- レポート化
    

---

## 4.2 TradingViewに寄せる理由

TradingView を活用することで、**チャート表示・アラート発火・Pine による一次検証** などの汎用機能を自前実装せずに済むため、北極星は独自価値の実装に集中できる。

特に削減できる実装対象は以下。

- 高機能チャート UI の自前実装
    
- テクニカル指標表示機能の構築
    
- アラート発火基盤の構築
    
- Pine Script による一次検証環境の構築
    
- スクリーナーやファンダ表示 UI の構築
    

ただし、**TradingView の有料プランで増えるのは主に TradingView 内での利用権限であり、埋め込み widget 上のデータ条件まで自動的に広がるわけではない**。 また、北極星が必要とする知見資産や独自導線は北極星側で保持する。

---

## 4.3 注意点

TradingViewは中核UIとして有用だが、**北極星の資産をTradingView側に依存させすぎない**ことが重要である。

北極星側で正本として保持すべき情報は以下とする。

- 監視対象の定義
    
- AI要約結果
    
- 銘柄研究ノート
    
- 投資仮説
    
- ルール定義
    
- バックテスト履歴
    
- レポート

### `補足`
ホームで扱う「マーケット概況」「注目イベント」「朝サマリ / 夜サマリ」は、画面仕様だけでなくデータ供給方式を定義して成立させる。 MVPでは以下を前提とする。
- market_snapshots を北極星側で定期取得・保存する
    
- key_events は alert_events / external_references / 決算イベント等から生成する
    
- 朝サマリ / 夜サマリは日本市場営業日基準で集約する
    
- 情報不足時は summary を空にせず、insufficient_context を明示する

## 10. 想定される利用フロー

### 10.1 毎日の利用フロー

1. ホームを開く
    
2. 朝サマリまたは夜サマリを確認する
    
3. 発火アラートとAI要約を読む
    
4. 気になる銘柄を銘柄詳細ページで深掘りする
    
5. 仮説ノートに追記する
    

### 10.2 銘柄研究フロー

1. 気になる銘柄を開く
    
2. チャート・ファンダ・ニュースを確認する
    
3. AI論点カードで整理された要点を把握する
    
4. 比較対象銘柄と比較する
    
5. 仮説ノートを作成・更新する
    

### 10.3 ルール検証フロー

1. 自然言語でルールを入力する
    
2. Pine Script に変換する
    
3. TradingViewで一次検証する
    
4. CSVを北極星に取り込む
    
5. レポートと比較結果を保存する

## 12. 現時点での結論

北極星は、TradingViewの強みを活用しつつ、北極星独自の価値である「AIによる意味づけ」「研究の蓄積」「検証の再利用性」に集中する方針で進める。

特に重要な判断は以下の通り。

- TradingView連携は **B案（実用本命）** を採用する
    
- TradingViewには **表示・監視・一次検証** を担当させる
    
- 北極星には **要約・比較・仮説・履歴管理** を担当させる
    
- MVPでは、3つのコンセプトが成立する最小機能に集中する
    

この方針により、短期間で実用性の高い個人用株価分析ツールを構築しつつ、将来的な拡張にも耐えられる設計を目指す。

## 14. MVP画面遷移

MVP時点の基本導線は、以下の通りとする。

### 14.1 毎日の確認導線

ホーム → 銘柄詳細 → 研究ノート

- ホームでアラートと要約を確認する
    
- 気になる銘柄を銘柄詳細で深掘りする
    
- 必要に応じて研究ノートを更新する
    

### 14.2 銘柄研究導線

ホーム → 銘柄詳細 → 比較ページ → 研究ノート

- 銘柄詳細で1銘柄の状況を把握する
    
- 比較ページで他銘柄との差分を確認する
    
- 結果を研究ノートに反映する
    

### 14.3 ルール検証導線

ルール検証ラボ → TradingViewで一次検証 → 北極星へCSV取込 → 検証履歴詳細

- 北極星で自然言語ルールを作成する
    
- Pine Scriptを生成する
    
- TradingViewで一次検証する
    
- CSVを北極星に取り込む
    
- 検証履歴に保存する
    

### 14.4 画面遷移の原則

- ホームは全機能の起点とする
    
- 銘柄詳細は「研究ノート」「比較ページ」へのハブとする
    
- ルール検証ラボは、検証履歴詳細と対になる独立ワークスペースとする
    
- TradingViewは北極星内画面の一部として利用するが、北極星の主導導線は北極星側で持つ

## 16. MVP情報設計上の重要判断

### 16.1 MVPでは画面数を増やしすぎない

MVPでは「ホーム」「銘柄詳細」「比較」「研究ノート」「ルール検証」「検証履歴」の6画面に抑える。

### 16.2 銘柄詳細を中核ハブにする

日々の利用でも研究でも、最終的には銘柄単位の判断に集約されるため、銘柄詳細画面を中核に置く。

### 16.3 ノートと検証履歴を資産化する

北極星の独自価値は、情報そのものよりも、情報に対する解釈と履歴の蓄積にあるため、ノートと検証履歴は早期に設計を固める。

### 16.4 TradingViewは画面部品として扱う

TradingViewを主導画面にせず、北極星内のコンポーネントとして埋め込むことで、主導権を北極星側に維持する。
